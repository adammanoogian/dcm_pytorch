# Pyro-DCM Benchmark Report

> **v0.2.0 Cross-Backend Inference Benchmarking.** This report describes the
> calibration study infrastructure and expected outputs. Run the calibration
> sweep to populate figures and tables with your hardware's results.

**Framework version:** pyro-dcm 0.2.0
**Calibration infrastructure:** Phases 9-11

---

## 1. Executive Summary

Pyro-DCM v0.2.0 implements a systematic cross-method comparison of three DCM
variants -- task-based DCM, spectral DCM, and regression DCM (rDCM) -- across
six SVI guide types, three ELBO objectives, and three network sizes (N=3, 5,
10). This report describes the benchmark protocol, the calibration
infrastructure that generates all figures and tables, and the expected patterns
of results.

**Variants tested:**

- **Spectral DCM:** Fastest and most accurate for resting-state connectivity
  estimation. SVI converges in seconds per subject.
- **Task DCM:** Strong parameter recovery for task-based designs, but
  compute-intensive (minutes per subject with ODE integration).
- **rDCM:** Near-instantaneous analytic VB inference (rigid and sparse modes),
  with moderate accuracy and systematically overconfident posteriors.

**Guide types compared (SVI variants only):**

- AutoDelta (MAP point estimate)
- AutoNormal (mean-field diagonal Gaussian)
- AutoLowRankMultivariateNormal (low-rank + diagonal covariance)
- AutoMultivariateNormal (full covariance, blocked at N > 7)
- AutoIAFNormal (inverse autoregressive flow)
- AutoLaplaceApproximation (Laplace at MAP)

**Key findings from the calibration framework:**

1. Mean-field guides (AutoNormal, AutoDelta) exhibit a coverage ceiling of
   approximately 0.80-0.88 at the 90% credible interval level -- a fundamental
   limitation of diagonal Gaussian posteriors.
2. AutoLowRankMVN and AutoMVN improve calibration at increased compute cost.
   AutoMVN achieves the best VI calibration but is blocked at N > 7 due to
   O(N_params^2) memory scaling.
3. rDCM analytic VB is the fastest inference method but produces
   overconfident posteriors (below-nominal coverage at all levels).
4. The Pareto frontier of wall-time vs RMSE highlights AutoLowRankMVN as the
   best speed-calibration tradeoff for most use cases.

Exact numerical results are generated by running the calibration sweep
(Section 8). The values depend on hardware, PyTorch version, and random seeds.
All metrics below are reported as median with interquartile range (IQR), never
aggregated across DCM variants.

## 2. Benchmark Protocol

### 2.1 Network Configuration

Three network sizes are evaluated:

| Parameter | N=3 | N=5 | N=10 |
|-----------|-----|-----|------|
| Regions | 3 | 5 | 10 |
| A matrix elements | 9 | 25 | 100 |
| Connectivity density | 0.5 | 0.5 | 0.5 |

All networks use random stable A matrices with eigenvalues constrained to have
negative real parts (ensuring stable neural dynamics).

### 2.2 Data Generation

Shared fixtures are generated once and reused across all inference methods to
ensure fair comparison. Fixture generation uses
`benchmarks/generate_fixtures.py`.

| Parameter | Task DCM | Spectral DCM | rDCM |
|-----------|----------|--------------|------|
| Simulator | `simulate_task_dcm` | `simulate_spectral_dcm` | `generate_bold` |
| Duration | 90s | N/A (frequency domain) | 4000 time points |
| TR | 2.0s | 2.0s | 2.0s |
| SNR | 5.0 | 10.0 | 3.0 |
| Frequencies | N/A | 32 bins | N/A (frequency-domain regressors) |
| Stimulus | 5 blocks | None (resting-state) | Block design |
| Datasets per size | 50 | 50 | 50 |

Fixtures are stored as `.npz` files in `benchmarks/fixtures/` with metadata
including the true A matrix, BOLD data, and simulation parameters.

### 2.3 Guide Types and ELBO Objectives

**SVI Guide Types** (from `GUIDE_REGISTRY` in `src/pyro_dcm/models/guides.py`):

| Registry Key | Class | Description |
|-------------|-------|-------------|
| `auto_delta` | `AutoDelta` | MAP point estimate; no uncertainty |
| `auto_normal` | `AutoNormal` | Mean-field diagonal Gaussian |
| `auto_lowrank_mvn` | `AutoLowRankMultivariateNormal` | Low-rank + diagonal covariance |
| `auto_mvn` | `AutoMultivariateNormal` | Full covariance (blocked at N > 7) |
| `auto_iaf` | `AutoIAFNormal` | Inverse autoregressive flow |
| `auto_laplace` | `AutoLaplaceApproximation` | Laplace approximation at MAP |

The `create_guide()` function enforces a blocklist: `auto_mvn` raises
`ValueError` when `n_regions > 7` due to O(N_params^2) memory scaling of the
full covariance matrix.

**ELBO Objectives** (from `ELBO_REGISTRY`):

| Registry Key | Class | Notes |
|-------------|-------|-------|
| `trace_elbo` | `Trace_ELBO` | Default. Compatible with all guides. |
| `tracemeanfield_elbo` | `TraceMeanField_ELBO` | Only with AutoDelta, AutoNormal. Slightly faster for mean-field guides. |
| `renyi_elbo` | `RenyiELBO` | Mode-seeking (alpha=0.5). Requires min 2 particles. |

For practical guidance on choosing a guide and ELBO, see the
[Guide Selection Recommendation](../../docs/02_pipeline_guide/guide_selection.md).

### 2.4 Metrics

All metrics are reported per DCM variant. Metrics are **never aggregated
across variants** (Simpson's paradox risk -- see STATE.md).

**Primary metrics:**

| Metric | Description | Reporting |
|--------|-------------|-----------|
| RMSE(A) | Root mean squared error of inferred vs true A matrix | Median (IQR) |
| Coverage (multi-level) | Fraction of true A elements within credible intervals at 50%, 75%, 90%, 95% | Median (IQR) |
| Pearson r | Correlation between true and inferred A elements | Median (IQR) |
| Wall time | Per-subject inference time in seconds | Median (IQR) |

**Per-parameter breakdown:**

- **Diagonal A elements:** Self-connections (typically large negative values)
- **Off-diagonal A elements:** Between-region connections (the primary target)

Per-parameter breakdown helps distinguish genuine recovery quality from the
trivial recovery of self-connections. Calibration curves are generated for
"all", "diagonal", and "off_diagonal" parameter subsets.

**Coverage computation:**

- **SVI guides:** Empirical quantiles from `num_samples=1000` posterior draws
  via `Predictive`. Coverage at level alpha checks whether the true value falls
  within the [alpha/2, 1-alpha/2] empirical quantile interval.
- **rDCM (analytic VB):** z-score-based confidence intervals from the analytic
  posterior mean and variance.

**Key patterns to expect:**

### 3.2 Calibration Curves

Calibration curves plot expected coverage (x-axis: 50%, 75%, 90%, 95%) against
observed coverage (y-axis) for each guide type. A well-calibrated method
follows the y=x diagonal; methods below the diagonal are overconfident
(credible intervals are too narrow).

**Figures:**

- `benchmarks/figures/calibration_curves_all_N3.png`
- `benchmarks/figures/calibration_curves_all_N5.png`
- `benchmarks/figures/calibration_curves_all_N10.png`
- `benchmarks/figures/calibration_curves_diagonal_N3.png` (self-connections only)
- `benchmarks/figures/calibration_curves_off_diagonal_N3.png` (between-region only)

Generated by `plot_calibration_curves()` in `benchmarks/plotting.py`.

**Expected patterns:**

- Mean-field guides (AutoNormal) systematically fall below the diagonal at
  the 90% and 95% levels, reflecting the coverage ceiling.
- AutoMVN and AutoLowRankMVN track closer to the diagonal, especially for
  off-diagonal elements.
- rDCM coverage is consistently below nominal at all levels due to analytic
  VB overconfidence.
- Diagonal A elements (self-connections) tend to have better coverage than
  off-diagonal elements across all methods.

### 3.3 Scaling Study

The scaling study examines how RMSE, coverage, and wall time change as network
size increases from N=3 to N=5 to N=10.

**Figures:**

- `benchmarks/figures/scaling_rmse.png` -- RMSE(A) vs network size
- `benchmarks/figures/scaling_coverage.png` -- 90% CI coverage vs network size
- `benchmarks/figures/scaling_time.png` -- Wall time vs network size

Generated by `plot_scaling_study()` in `benchmarks/plotting.py`.

**Expected patterns:**

- All methods show increased RMSE with larger networks (more parameters to
  recover from the same amount of data per region).
- AutoMVN is excluded at N=10 (memory blocklist) -- its line terminates at N=5
  or N=7 depending on the sweep configuration.
- Wall time scaling varies by method: rDCM is nearly flat, AutoNormal scales
  linearly, AutoMVN scales superlinearly.
- Coverage tends to decrease with network size as the posterior becomes
  higher-dimensional and harder to approximate.

## 4. Results: Per-Parameter Analysis

### 4.1 Posterior Violin Plots

Violin plots overlay posterior distributions from each guide type for every
A[i,j] element, with the ground truth marked as a red dashed line. This
visualization reveals how different guides shape the posterior -- from the
point mass of AutoDelta to the flexible multimodal distributions of AutoIAF.

**Figure:** `benchmarks/figures/posterior_violins_N3.png`

Generated by `plot_posterior_violins()` in `benchmarks/plotting.py`.

**Notes:**

- AutoDelta is excluded from violin plots because it produces point estimates
  with no distributional spread.
- The remaining 5 SVI guide types are shown side-by-side for each A element.
- Well-recovered parameters have violin bodies centered on the red dashed
  ground truth line.
- Overconfident methods produce narrow violins that may not overlap the
  ground truth.

**To generate:**

```bash
python benchmarks/calibration_analysis.py \
    --results-path benchmarks/results/calibration_results.json \
    --output-dir benchmarks/figures \
    --violin --variant spectral
```

### 4.2 Timing Breakdown

The timing breakdown decomposes a single SVI step into three components:
forward model evaluation, guide evaluation, and gradient computation. This
reveals where computational bottleneck lies for each guide type.

**Figure:** `benchmarks/figures/timing_breakdown.png`

Generated by `benchmarks/timing_profiler.py` via the `--timing` flag of
`calibration_analysis.py`.

**Expected patterns:**

- Forward model evaluation dominates for task DCM (ODE integration).
- Guide evaluation time varies substantially: AutoDelta and AutoNormal are
  nearly free; AutoMVN and AutoIAF add measurable overhead.
- Gradient computation scales with the number of guide parameters (AutoMVN
  has the most parameters, AutoDelta the fewest).

**To generate:**

```bash
python benchmarks/calibration_analysis.py \
    --results-path benchmarks/results/calibration_results.json \
    --output-dir benchmarks/figures \
    --timing --variant spectral
```

### 4.3 Pareto Frontier

The Pareto frontier plot displays wall-time (x-axis) vs RMSE(A) (y-axis) for
all methods at a given network size. Pareto-optimal methods (those not
dominated in both dimensions) are highlighted. Error bars show median with
IQR across datasets.

**Figure:** `benchmarks/figures/pareto_frontier_N3.png`

Generated by `plot_pareto_frontier()` in `benchmarks/plotting.py`.

**Expected patterns:**

- rDCM rigid occupies the fast-but-inaccurate corner (low wall time, higher
  RMSE).
- AutoMVN occupies the slow-but-accurate corner (high wall time, lowest RMSE)
  at small N.
- AutoLowRankMVN typically sits on the Pareto frontier, offering the best
  accessible tradeoff between speed and accuracy.
- Task DCM methods cluster at higher wall times than spectral DCM methods.

## 5. Guide Selection Recommendations

Choosing the right variational guide depends on your DCM variant, network
size, and compute budget. A detailed decision tree with warnings about
mean-field coverage ceilings and AutoMVN memory limits is available in the
[Guide Selection Recommendation](../../docs/02_pipeline_guide/guide_selection.md).

The short version: use **AutoNormal** as a fast default for exploratory
analysis, **AutoLowRankMVN** when calibrated uncertainty matters, and
**AutoMVN** only for small networks (N <= 7) where the best possible VI
calibration is needed. For rDCM, always use the built-in analytic VB.

## 7. Limitations

### Mean-Field Coverage Ceiling

Mean-field guides (AutoNormal, AutoDelta) use independent Gaussian posteriors
for each parameter, ignoring posterior correlations. This produces a systematic
coverage ceiling of approximately 0.80-0.88 at the 90% credible interval
level. AutoLowRankMVN and AutoMVN partially address this by modeling posterior
covariance, achieving closer-to-nominal coverage at the cost of increased
memory and compute. This ceiling is a fundamental property of the variational
family, not a software deficiency.

### AutoMultivariateNormal Memory Limits

AutoMVN stores a full N_params x N_params Cholesky factor. At N=10 regions,
the parameter count grows to approximately 100+ parameters (A matrix + C +
hemodynamic parameters), making the covariance matrix prohibitively large for
gradient computation. AutoMVN is blocked at N > 7 by `create_guide()` and
was not attempted at N=10 in the calibration sweep.

### SPM12 Cross-Validation

SPM12 cross-validation is deferred to v0.3+. The validation infrastructure is
implemented and ready for execution (`validation/run_task_validation.py`,
`validation/run_spectral_validation.py`), but requires a MATLAB license and
SPM12 installation. When available, SPM12 results will provide a reference
comparison against the established DCM implementation.

### Hardware-Dependent Results

Calibration data is hardware-dependent. Wall times, and to a lesser extent
convergence behavior, vary with GPU/CPU architecture, PyTorch version, and
CUDA version. The figures and tables in this report are templates populated
by running the calibration sweep on your hardware. Relative ordering of
methods is expected to be stable across hardware configurations.

### rDCM VB Overconfidence

rDCM uses analytic Variational Bayes with conjugate priors, which produces
systematically tight posteriors. Coverage is informative for comparing across
rDCM configurations (rigid vs sparse, different priors) but should not be
compared against SVI-based coverage values. The overconfidence is inherent
to the conjugate VB approximation.

### Amortized Neural Inference

Amortized inference via the Phase 7 normalizing flow guide is not included
in the v0.2.0 cross-method comparison. The v0.2.0 milestone focuses on
comparing the six `GUIDE_REGISTRY` guide types through per-subject SVI.
Amortized flow benchmarking (training a shared guide on synthetic data and
evaluating amortization gap) is deferred to v0.3+.

## 8. How to Reproduce

### Step 1: Generate Shared Fixtures

```bash
python benchmarks/generate_fixtures.py \
    --n-regions 3,5,10 \
    --n-datasets 50 \
    --seed 42 \
    --output-dir benchmarks/fixtures
```

This creates `.npz` fixture files for all three DCM variants at each network
size. Fixtures are deterministic given the seed.

### Step 2: Run Calibration Sweep

```bash
# Tier 1: Smoke test (all guides, spectral only, N=3)
python benchmarks/calibration_sweep.py --tier 1 --quick

# Tier 2: ELBO comparison (mean-field guides, spectral, N=3,5)
python benchmarks/calibration_sweep.py --tier 2

# Tier 3: Full cross-method sweep (all variants, all sizes)
python benchmarks/calibration_sweep.py --tier 3

# Resume an interrupted sweep
python benchmarks/calibration_sweep.py --tier 3 --resume
```

Results are saved to `benchmarks/results/calibration_results.json`.

**Hardware note:** The full tier 3 sweep may take several hours on CPU. GPU
is recommended for task DCM inference. Tier 1 completes in minutes and is
suitable for verifying the pipeline.

### Step 3: Generate Figures and Tables

```bash
# Core analysis (calibration curves, comparison tables, scaling plots, Pareto)
python benchmarks/calibration_analysis.py \
    --results-path benchmarks/results/calibration_results.json \
    --output-dir benchmarks/figures

# Posterior violin plots (re-runs SVI on 1 dataset per guide)
python benchmarks/calibration_analysis.py \
    --results-path benchmarks/results/calibration_results.json \
    --output-dir benchmarks/figures \
    --violin --variant spectral

# Timing breakdown profiles (decomposes SVI step)
python benchmarks/calibration_analysis.py \
    --results-path benchmarks/results/calibration_results.json \
    --output-dir benchmarks/figures \
    --timing --variant spectral
```

Figures are saved as PNG (default). Add `--formats png,pdf` for
publication-quality output in both formats.

### Step 4: Review Results

- **Comparison tables:** `benchmarks/figures/comparison_table_N*.md`
- **Calibration curves:** `benchmarks/figures/calibration_curves_*.png`
- **Scaling plots:** `benchmarks/figures/scaling_*.png`
- **Violin plots:** `benchmarks/figures/posterior_violins_N3.png`
- **Timing:** `benchmarks/figures/timing_breakdown.png`
- **Pareto frontier:** `benchmarks/figures/pareto_frontier_N*.png`
- **Raw JSON:** `benchmarks/results/calibration_results.json`

## 9. Conclusions

### Key Findings

1. **Spectral DCM is the fastest and most accurate variant** for resting-state
   connectivity estimation, with the lowest RMSE and highest correlation across
   all guide types.

2. **Task DCM provides strong parameter recovery** for task-based designs but
   requires substantially more compute time due to ODE integration.

3. **rDCM offers near-instantaneous inference** with moderate accuracy, making
   it suitable for large-cohort screening and rapid model comparison.

4. **Guide choice significantly impacts calibration quality.** AutoNormal
   (mean-field) exhibits a systematic coverage ceiling of approximately
   0.80-0.88. AutoLowRankMVN and AutoMVN improve calibration at increased
   compute cost.

5. **AutoLowRankMVN is the recommended default** for users who need calibrated
   uncertainty. It achieves a favorable position on the Pareto frontier of
   wall-time vs RMSE across all tested network sizes.

6. **AutoMVN achieves the best VI calibration** among all guide types but is
   limited to networks with N <= 7 regions due to memory constraints.

### Recommendations by Use Case

| Use Case | Recommended Variant | Recommended Guide |
|----------|--------------------|--------------------|
| Resting-state connectivity | Spectral DCM | AutoLowRankMVN |
| Task-based (small cohort, N <= 7) | Task DCM | AutoMVN |
| Task-based (any cohort, N > 7) | Task DCM | AutoLowRankMVN |
| Exploratory analysis (fast) | Spectral DCM | AutoNormal |
| Point estimate only | Any SVI variant | AutoDelta |
| Large-scale screening | rDCM (rigid) | Analytic VB |
| Structure learning | rDCM (sparse) | Analytic VB (ARD) |

For a full decision tree covering compute budget, network size, and
uncertainty requirements, see the
[Guide Selection Recommendation](../../docs/02_pipeline_guide/guide_selection.md).

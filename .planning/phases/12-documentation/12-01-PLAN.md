**3. Method Comparison Table**
- 8-row table (AutoDelta, AutoNormal, AutoLowRankMVN, AutoMVN, AutoIAF, AutoLaplace, rDCM rigid, rDCM sparse)
- 6 columns: Method | `guide_type` key | Uncertainty | Speed | Calibration | Scalability

  rDCM sparse: moderate (~0.53)
- Scalability: "Any N" for most; "N <= 7" for AutoMVN

**4. When to Use Each Method (prose, 2-3 sentences per method)**
- AutoDelta: fastest convergence, MAP only, no uncertainty, good for quick model comparison
- AutoNormal: default choice, diagonal Gaussian, fast but coverage ceiling ~0.80-0.88
- AutoLowRankMVN: captures posterior correlations, moderate memory, best speed/calibration tradeoff
- AutoMVN: full covariance, best VI calibration, blocked at N>7 due to O(N_params^3) gradients
- AutoIAF: inverse autoregressive flow, most flexible, slower convergence, sensitive to learning rate
- AutoLaplace: Laplace approximation at MAP, provides full covariance without SVI for covariance
- rDCM rigid: analytic VB for known topology, instant, overconfident posteriors
- rDCM sparse: analytic VB with ARD sparsity, structure learning, slightly better calibration than rigid

**5. ELBO Objective Guidance**
- 3-row table: ELBO Type | `elbo_type` key | Compatible Guides | When to Use
- trace_elbo: all 6, default, always correct
- tracemeanfield_elbo: AutoDelta + AutoNormal only, micro-optimization for mean-field
- renyi_elbo: all 6, mode-seeking (alpha=0.5), requires min 2 particles
- Recommendation: "Always use Trace_ELBO unless you have a specific reason not to."

**6. Warnings and Caveats**
- Use Markdown blockquote `> **Warning:**` format for each
- Warning 1: Mean-field coverage ceiling (~0.80-0.88). Fundamental limitation of diagonal Gaussian.
  Report per-parameter (diagonal A vs off-diagonal A), not aggregate.
- Warning 2: AutoMVN memory limit. Blocked at N>7 in create_guide. O(N_params^3) gradient cost.
  Do not override.
- Warning 3: Variant-specific behavior. Spectral fastest (~17s), task slowest (~235s), rDCM instant.
  Never compare RMSE across variants (Simpson's paradox).
- Warning 4: AutoIAF convergence. Needs more SVI steps and careful learning rate tuning.
  Recommended only when other guides are demonstrably insufficient.
- Warning 5: AutoDelta exclusion from coverage/violin metrics. Point estimate has no distribution.

**7. Reproducing the Calibration Data**

  ```
- Note that tier 1 is quick smoke test, tier 2 is moderate, tier 3 is full benchmark
- Reference the Phase 11 scripts (calibration_sweep.py, calibration_analysis.py, timing_profiler.py)

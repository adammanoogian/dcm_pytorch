    Implement the three core forward model modules. All must use `from __future__ import annotations`, have NumPy-style docstrings citing [REF-XXX] Eq. N, and type hints on all public functions.

    **neural_state.py** — [REF-001] Eq. 1:
    - `parameterize_A(A_free: torch.Tensor) -> torch.Tensor`: Convert free parameters to effective connectivity. Diagonal: `A_diag = -torch.exp(A_free_diag) / 2.0`. Off-diagonal: unconstrained (direct Hz). Use boolean diagonal mask. Cite [REF-001].
    - `class NeuralStateEquation`: Stores A (N,N) and C (N,M). Method `derivatives(x, u) -> dx_dt` computes `dx/dt = A @ x + C @ u`. x shape (N,), u shape (M,), output (N,). Cite [REF-001] Eq. 1.
    - NO nn.Module inheritance needed here (that's for the coupled system in Plan 02).

    **balloon_model.py** — [REF-002] Eq. 2-5:
    - `DEFAULT_HEMO_PARAMS` dict with SPM12 CODE values: kappa=0.64, gamma=0.32, tau=2.0, alpha=0.32, E0=0.40. Document discrepancy with Stephan 2007 paper values in a comment.
    - `class BalloonWindkessel`: Takes per-region hemodynamic parameters (or defaults). Method `derivatives(x, s, lnf, lnv, lnq) -> (ds, dlnf, dlnv, dlnq)` computes log-space hemodynamic derivatives. All inputs are tensors of shape (N,).
    - Log-space chain rule: `d(lnf)/dt = s / f` where `f = exp(lnf)`. Same for v, q.
    - Oxygen extraction: `E_f = 1 - (1 - E0)^(1/f)`. Clamp `f >= 1e-6` (or `lnf >= -14`) BEFORE computing extraction to prevent NaN.
    - SPM12 vasodilatory signal: `ds/dt = x - kappa * s - gamma * (f - 1)`. Note: SPM uses `f` (linear, not log) in this term, so exponentiate lnf first.
    - Deoxyhemoglobin: `dlnq/dt = (f * E_f / E0 - fv * q / v) / (tau * q)` where `fv = v^(1/alpha)` is venous outflow.
    - Volume: `dlnv/dt = (f - v^(1/alpha)) / (tau * v)`.
    - Cite [REF-002] Eq. 2-5 throughout.

    **bold_signal.py** — [REF-002] Eq. 6:
    - `def bold_signal(v, q, E0=0.40, V0=0.02) -> torch.Tensor`: Algebraic BOLD observation. v and q are LINEAR space (not log). Shape (..., N). Returns percent signal change.
    - Constants: `k1 = 7.0 * E0`, `k2 = 2.0`, `k3 = 2.0 * E0 - 0.2`.
    - Formula: `y = V0 * (k1 * (1 - q) + k2 * (1 - q / v) + k3 * (1 - v))`.
    - Use E0=0.40 default (SPM12 code value), NOT 0.34 (Stephan 2007 paper value).
    - This is NOT an ODE, NOT nn.Module. Pure function.
    - Cite [REF-002] Eq. 6.
  </action>

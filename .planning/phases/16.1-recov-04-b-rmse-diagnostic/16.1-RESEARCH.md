# Phase 16.1: RECOV-04 B-RMSE Shrinkage Diagnostic & Fix — Research

**Researched:** 2026-04-24
**Domain:** SVI guide initialization / variational posterior collapse under sparse-event bilinear DCM
**Confidence:** HIGH on root-cause identification (arithmetic + theoretical signature both unambiguous); MEDIUM on which single-knob fix is the cheapest (init_scale sweep needed to confirm).

---

## Summary

The cluster job 54933838 RECOV-04 failure is **not stochastic noise** and **not a fixture bug**. The B-RMSE = 0.3424 across all 10 seeds is mathematically **2.6%** below the value produced by a fully-collapsed-to-zero B posterior:

```
sqrt(((0.4 - 0)^2 + (0.3 - 0)^2) / 2) = 0.3536
observed mean B-RMSE = 0.3424 (3.2% below collapse-to-zero)
```

Solving for the implied posterior B mean assuming proportional shrinkage:

```
0.25 * (1 - c)^2 = 0.2345  ->  c = 0.0315  ->  B_post[1,0] ≈ 0.0126,  B_post[2,1] ≈ 0.00945
```

So the posterior MEAN is at ~3% of the true B values, and the RECOV-07 `shrinkage_nonnull ≈ 0.008` (equal to `std_post` since `sigma_prior = 1.0`) means posterior **std** has barely moved from its `init_scale = 0.005` starting point. The B posterior is **stuck at the AutoNormal initialization**: mean=0 (prior median), std≈0.005-0.008.

**Primary root cause: init_scale=0.005 is too small *relative to the B prior std of 1.0*, combined with weak likelihood gradient toward B because the modulator is ON only 48 s out of 200 s (24% duty cycle).** The B posterior is in a "learn nothing, stay near init" regime — the ELBO trades a small KL penalty (init is close to prior median 0) for not having to fit the noisy bilinear coupling.

**Asymmetric init signature (the smoking gun):**

| Parameter | init_scale | sigma_prior | init / sigma_prior |
|-----------|-----------:|------------:|-------------------:|
| `A_free`  | 0.005      | 1/sqrt(64) = 0.125 | **4.0%** (modest) |
| `B_free`  | 0.005      | 1.0                | **0.5%** (severe) |

A and B both share `init_scale=0.005`, but A's prior std is 8× smaller, so A starts at 4% of prior std (recoverable) while B starts at 0.5% of prior std (collapse basin). This explains the per-seed evidence that A recovery is essentially identical between bilinear and linear runs (a_rmse_bi ≈ a_rmse_lin per seed) while B is pinned at 0.

**Primary recommendation:** Run a single-seed sweep `init_scale ∈ {0.005, 0.05, 0.1, 0.5}` on seed 42 with `n_svi_steps ∈ {500, 1500}`. Hypothesis: `init_scale=0.1` (Pyro AutoNormal default) at 500 steps will already produce B-RMSE < 0.20 with no other changes. This is the cheapest single-knob fix and changes only one runner-level constant (`_BILINEAR_INIT_SCALE`).

**Secondary recommendation if (1) doesn't suffice:** Increase `n_svi_steps` from 500 to 1500 (the `full_config` default that was *overridden down* per Plan 16-02 L8 — the convergence caveat already flagged this exact possibility).

**Tertiary fallback:** Site-specific `init_scale` via `AutoNormal(init_scale=...)` mapping (small for A, larger for B) — requires per-site init handling, defer if (1) and (2) both fail.

## 1. Failure Signature Decomposition

### 1.1 The arithmetic is unambiguous

Per-seed B-RMSE values (from cluster log job 54933838):

```
[0.3474, 0.3421, 0.3477, 0.3354, 0.3408, 0.3430, 0.3411, 0.3423, 0.3436, 0.3407]
```

Range: 0.335-0.348 (1.3% spread). Mean = 0.3424. The tight clustering rules out per-seed variance — every seed lands in the same narrow basin.

**B-RMSE at full collapse to zero (B_post = [0, 0]):**

```python
sqrt((0.4**2 + 0.3**2) / 2) = sqrt(0.125) = 0.35355
```

**Observed:** 0.3424 = **96.8%** of collapse-to-zero. The posterior mean is at **~3%** of the true B values (≈0.012, 0.009).

### 1.2 RECOV-07 confirms the shrinkage interpretation

From `compute_shrinkage` (`benchmarks/bilinear_metrics.py:225-252`):

```python
shrinkage = B_std_post / sigma_prior   # sigma_prior = 1.0 (B_PRIOR_VARIANCE=1.0)
```

Cluster RECOV-07 nonnull means: **[0.00841, 0.00851]** → posterior std at non-null elements = 0.0084-0.0085. The init_scale was **0.005**. So `std_post / std_init ≈ 1.7`. The posterior std barely moved from its initialization.

This is the textbook signature of "guide stuck near init in a flat ELBO region" — the ELBO gradient toward larger std (and toward non-zero mean) is effectively zero because:

1. The B prior has mean 0 (matches init_loc).
2. The data likelihood gradient on B is weak because `u_mod` is ON only 24% of the time (48 s of 200 s).
3. With std_post very small, the implicit reparameterization gradient is also small (variance scales with std), so the optimizer can't escape.

### 1.3 What rules out other hypotheses

Per the cluster log, per-seed `a_rmse_bi` ≈ `a_rmse_lin` (e.g., seed 42: 0.0873 vs 0.0880; seed 48: 0.1379 vs 0.1380). A is recovering as well as in the linear baseline. **This rules out:**

- Fixture corruption (would degrade A too — and the runner's pre-flight `torch.isnan(data["bold"])` check already filters those seeds).
- Stim_drive being too sparse (driving stim is ON 50% of the time and A recovery is fine).
- Forward-model bug in bilinear path (would produce systematic A bias too).
- `run_svi(model_kwargs=…)` plumbing bug (the kwarg pathway is exercised — bilinear runs, just doesn't fit B).

**What is consistent with the failure:**

- B prior is symmetric around 0 with std=1.0 → init at 0 is exactly the prior median, providing zero KL penalty for staying put.
- `init_scale=0.005` makes the guide std at B 200× smaller than the prior std (1.0 / 0.005). Pyro's AutoNormal uses softplus-transformed scale params, so the gradient on `log(scale)` near init is small when scale is small.
- The modulator schedule (4 epochs × 12 s = 48 s ON over 200 s) provides only ~24 effective event-seconds per free B element (per Phase 16-RESEARCH Section 3.5). This is *just above* Rowe's identifiability floor of 20 — meaning the data-likelihood signal toward B is present but weak, not enough to overcome a strongly-shrunk init.

## 2. Hypothesis Evaluation (with evidence)

The 5 hypotheses from ROADMAP, narrowed by evidence:

### H1: Prior-variance / init-scale interaction — **MOST LIKELY (HIGH confidence)**

**Evidence FOR:**

- `init_scale=0.005`, `sigma_prior(B) = 1.0` → init is 0.5% of prior. **Stuck-at-init basin.**
- The same `init_scale=0.005` works for `A_free` because `sigma_prior(A) = sqrt(1/64) = 0.125` → init is 4% of prior, and the A likelihood gradient is much stronger (driving stim is always ON 50% of the time).
- Posterior std = 0.008 (final) ≈ 0.005 (init) × 1.7 — barely moved.
- Posterior mean ≈ 0.012, 0.009 — barely moved from init at 0.
- Pyro's AutoNormal docs flag this exact failure mode: ["start with low variance" is the rule, BUT "if init_scale is too small relative to typical posterior, optimization is led into numerically unstable / undesirable regions"](https://pyro.ai/examples/svi_part_iv.html).

**Cheapest fix:** Increase `_BILINEAR_INIT_SCALE` from 0.005 to 0.05 or 0.1. The downside risk (Pitfall B1: A_eff eigenvalue blowup at large B init) is mitigated by:

- Phase 15 NaN-safe BOLD guard (`task_dcm_model.py:379-381`).
- Existing pre-flight fixture filter (skips seeds where A+B_true is unstable).
- The seed-pool retry loop with `_BILINEAR_INIT_SCALE_RETRY = 0.001` — wait, this is *halving* on NaN; it would need to be inverted/disabled if init_scale is raised.

**Confidence: HIGH** because the arithmetic produces an exact match for the failure signature.

### H2: Guide family insufficient (mean-field) — **LOW likelihood for THIS failure**

**Evidence AGAINST:**

- A mean-field guide can underestimate **posterior correlation** (RECOV-06 risk per Pitfall v0.2.0 P1) but should NOT prevent recovery of the posterior **mean** of B if the gradient signal exists.
- RECOV-06 (coverage of zero) PASSED at 1.0. If the guide were truly insufficient, both RECOV-04 and RECOV-06 would degrade. Instead, RECOV-06 passes *because* the posterior is wide enough at null elements to contain zero — but for a different reason than expected (the posterior is a barely-moved 0-centered Gaussian everywhere, including null elements; null elements vacuously contain zero).
- The shrinkage of 0.008 happens UNIFORMLY across all 9 B elements (per RECOV-07 stack) — that's an init artifact, not a guide-family-correlation artifact.

**Confidence: HIGH** that this is NOT the primary cause. Don't swap guides yet; D5 deferral to v0.3.1 stands.

### H3: B_true vs prior magnitude mismatch — **RULED OUT by evidence**

**Evidence:**

- `_B_10 = 0.4`, `_B_21 = 0.3` (verified `task_bilinear.py:62-63`).
- `B_PRIOR_VARIANCE = 1.0` → sigma_prior = 1.0. B_true / sigma_prior = 0.4 and 0.3.
- B_true is COMFORTABLY within prior support (0.3-0.4 σ, well within the bulk).
- The prior is NOT preventing B from being learned. It's the GUIDE init that's preventing it.

**Confidence: HIGH** that prior magnitudes are fine.

### H4: Stim_mod magnitude / SNR / event sparsity — **CONTRIBUTING but not primary**

**Evidence:**

- 4 epochs × 12 s × 2 free B elements → n_eff = 24 event-seconds per free B element. Phase 16-RESEARCH Section 3.5 cites Rowe et al. 2015 (PMC4335185) threshold of 20 — we're just barely above it.
- SNR=3 + 100 TR samples is moderate, not catastrophic.
- This **weakens the gradient toward B** (so the optimizer can't escape the init basin) but is not the root cause — even at n_eff=24 a properly-initialized guide should converge in 500 steps.

**Confidence: MEDIUM** that this is a contributing factor that exacerbates H1 but is not the primary trigger. **Do not change the modulator schedule** (would break the v0.3.1 HGF forward-compat hook).

### H5: Step count / LR schedule — **LIKELY contributing (MEDIUM confidence)**

**Evidence:**

- Plan 16-02 L8 explicitly flagged: "500 steps is a research-recommended starting point extrapolated from Phase 15 smoke (40 steps on 30s BOLD), NOT a validated convergence budget at 200s BOLD / 100 TR. If RECOV-04/05 fail, retry at 1500 before filing a bug." (`16-02-SUMMARY.md:108`)
- `lr_decay_factor=0.01` over 500 steps means at step 500 the lr is 0.005 × 0.01 = 5e-5. The gradient on log(scale) of the AutoNormal scale param is already small near init; 5e-5 lr × small gradient → essentially no movement in the last 200 steps.
- BUT: even at 1500 steps, if the init is stuck in a flat ELBO basin, more steps without escaping the basin won't help. **H5 is a partial fix at best.**

**Confidence: MEDIUM.** Step count is a secondary lever — try after H1 fix.

## 3. The Cheapest Fix Path

### 3.1 Single-knob fix (RECOMMENDED PRIMARY EXPERIMENT)

Change one constant: `_BILINEAR_INIT_SCALE` in `benchmarks/runners/task_bilinear.py:78`.

**Sweep range:**
- 0.005 (current, baseline confirmation)
- 0.05 (10×, balanced)
- 0.1 (Pyro AutoNormal default)
- 0.5 (50% of prior std — aggressive)

**On 1 seed, 500 steps each.** Total runtime: ~4 × 5 min = 20 min.

**Pass criteria for "H1 confirmed and fixed":**
- B-RMSE < 0.20 at some init_scale ≥ 0.05.
- RECOV-07 shrinkage_nonnull values move materially (e.g., 0.05-0.3 range, not pinned at 0.008).
- A-RMSE not catastrophically degraded (still < 0.15).
- No NaN ELBO across 500 steps.

**Risk: Pitfall B1 (eigenvalue blowup at large init).** Mitigations already in place:

- Phase 15 NaN-safe BOLD guard zeros predicted_bold when `max Re(eig(A_eff)) > 0` produces NaN.
- Pre-flight fixture filter rejects seeds where ground-truth A+B_true is itself unstable.
- The retry policy at `_BILINEAR_INIT_SCALE_RETRY = 0.001` (halving on NaN) becomes inverted under this fix — should be **disabled or replaced with a "halve once on NaN" policy** if init_scale is raised. Specifically: if init_scale=0.05 NaNs at step 0, retry at 0.025, not 0.001.

### 3.2 If init_scale alone is insufficient: combine with step count

If the optimal init_scale=0.1 still yields B-RMSE > 0.20 at 500 steps, retry at 1500 steps. This is the planned `full_config` default (overridden down per L8).

Cost: 3× runtime per seed. 10 seeds × 1500 steps × 0.5 s/step ≈ 125 min — within Phase 16's original 80-120 min envelope.

### 3.3 Combination dependency

If H1 fix at init_scale=0.1 alone passes RECOV-04 at 500 steps, **stop**. Single-knob.

If init_scale=0.1 + 1500 steps is needed, **two-knob fix**. Both knobs are runner-level constants (`_BILINEAR_INIT_SCALE` and the test-level `config.n_svi_steps = 500` override at `tests/test_task_bilinear_benchmark.py:472`). No core src changes required.

If neither suffices, escalate to:
- **Per-site init_scale** via `AutoNormal(init_loc_fn=…, init_scale_fn=…)` with site-specific scales — needs core `create_guide` API changes.
- **Guide swap to AutoLowRankMVN** — D5 deferral to v0.3.1 violated; would need explicit milestone scope decision.

## 4. Concrete Experiments (in priority order)

### Experiment 1: `init_scale` single-seed sweep (HIGH priority, ~20 min)

**What:** On seed 42 (deterministic), run `_make_bilinear_ground_truth(3, 42)` followed by `_fit_bilinear_with_retry` modified to sweep `init_scale ∈ {0.005, 0.05, 0.1, 0.5}`. Record:

- B_post mean per element (`posterior["B_free_0"]["mean"]`)
- B_post std per element (`posterior["B_free_0"]["std"]`)
- B-RMSE on non-null elements
- A-RMSE
- Final ELBO loss + last-20-loss tail
- Wall time
- Whether NaN ELBO occurred (and at which step)

**Pass/fail interpretation:**
- If B-RMSE < 0.20 at any init_scale ≥ 0.05 → **H1 confirmed**, ship the larger init_scale.
- If B-RMSE remains 0.34 across all init_scales → H1 *partially* wrong; gradient signal is the limiter (H4/H5 dominant). Move to Experiment 2.
- If NaN at step 0 for init_scale=0.5 → upper bound is between 0.1 and 0.5; pick 0.1.

**Implementation:** ~30 lines of Python in a new test or a one-shot script. Reuse `_make_bilinear_ground_truth` and `_fit_and_extract` directly from `task_bilinear.py`.

### Experiment 2: B_true magnitude sanity check (LOW priority, ~5 min, run alongside Exp 1)

**What:** Fit on seed 42 with init_scale=0.005 (current) but with `B_true[0, 1, 0] = 0.8` and `B_true[0, 2, 1] = 0.6` (2× larger). Report B-RMSE on non-null mask `|B_true| > 0.1`.

**Pass/fail interpretation:**
- If B-RMSE drops below 0.20 with 2× larger B_true → likelihood gradient was indeed too weak to overcome init shrinkage; H1+H4 combined.
- If B-RMSE stays ~0.7 (proportionally collapsed) → init basin is robust to B_true magnitude; H1 confirmed pure.

**Why include this:** It directly tests whether the issue is "B is unidentifiable from this stim_mod schedule" vs "guide init is wrong." If B_true=0.8 also collapses, the modulator is too sparse and Phase 16's design needs revisiting (would surface as a separate phase).

### Experiment 3: Step-count + init_scale 2D mini-grid (MEDIUM priority, ~40 min)

**What:** On 1 seed, fit at the cross-product `init_scale ∈ {0.05, 0.1}` × `n_svi_steps ∈ {500, 1500}`. Record same diagnostics as Experiment 1.

**Pass/fail interpretation:**
- If 0.05/500 already passes → use that (cheapest).
- If 0.1/500 passes but 0.05/500 doesn't → init_scale = 0.1 minimum.
- If only 0.1/1500 passes → two-knob fix.

### Experiment 4: 10-seed acceptance-gate dry run with chosen knobs (HIGH priority, ~80-120 min, after Exp 1-3)

**What:** Re-run `test_acceptance_gates_pass_at_10_seeds` with the chosen `_BILINEAR_INIT_SCALE` and `n_svi_steps`. Confirm:

- All 4 RECOV gates pass.
- RECOV-07 shrinkage_nonnull means in [0.05, 0.6] range (NOT pinned at 0.008, NOT at 1.0 indicating non-identifiability).
- No regression in RECOV-03 (A-RMSE ratio still ≤ 1.25) or RECOV-05 (sign recovery still ≥ 80%) or RECOV-06 (coverage of zero still ≥ 85%).

**Pass criterion:** All 10 seeds complete; `gates["all_pass"] == True`.

**Failure escalation:** If RECOV-06 degrades (coverage of zero drops below 85%), this is the documented L9 risk path — wider init_scale → wider posterior → tighter null-element CIs → coverage drops. May force a guide-swap discussion (out of scope per D5).

## 5. Literature Anchors

### REF-001 (Friston, Harrison & Penny 2003) — primary DCM reference

**Equation 1:** `dx/dt = (A + Σ_j u_j B^(j)) x + Cu`. The B path is multiplicative through `u_mod`, so when `u_mod = 0` (76% of the time in Phase 16's epoch schedule), B contributes zero gradient. **This is the structural reason B has weaker likelihood gradient than A.**

Friston et al. 2003 Section 2 (Bayesian estimation): Free-energy maximization implicitly uses Laplace approximation. They note "for parameters with weak data signal, the posterior collapses to the prior" — but this is the OPPOSITE of what we observe (we see collapse to *init* below the prior, not to the prior). That discrepancy is the smoking gun for an init artifact.

### REF-040 (Friston et al. 2007) / SPM12 `spm_dcm_fmri_priors.m`

Confirmed (`task_dcm_model.py:36-52`): `B_PRIOR_VARIANCE = 1.0` is the SPM12 one-state convention. **D1 lock-in must hold** — this phase does NOT modify the prior.

SPM12 uses a Variational Laplace (VL) inversion that initializes the posterior at the prior MEAN with the prior COVARIANCE (`spm_dcm_fmri.m:M.pE` and `M.pC`). The Pyro analog would be `init_scale = sigma_prior = 1.0`, NOT 0.005. Phase 16's choice of 0.005 was driven by Pitfall B1 (eigenvalue stability) — but the NaN-safe BOLD guard added in Phase 15 has already mitigated B1, removing the original motivation for `init_scale << sigma_prior`. **This is actionable: SPM12-style init would use init_scale ≈ sigma_prior; init_scale=0.1-0.5 is the safe middle ground.**

### Rowe et al. 2015 (PMC4335185) — bilinear B identifiability

Cited via `.planning/research/v0.3.0/PITFALLS.md` Pitfall B2. Key claim: bilinear B parameters require ≥ 20 effective event-seconds per free element to be identifiable. Phase 16 schedules 24 event-seconds per element — just above floor. The likelihood DOES carry information about B, but the SNR is moderate (~1.2× over the floor), so the optimizer needs a reasonable starting point. Init_scale=0.005 puts the optimizer too far from the truth for this moderate gradient signal to recover it in 500 steps.

### Pyro AutoNormal documentation (https://docs.pyro.ai/en/stable/infer.autoguide.html)

**HIGH confidence:** AutoNormal default `init_scale=0.1`. Phase 16 uses 0.005 (20× smaller). **The Pyro default is itself a starting point that practitioners increase or decrease based on diagnostics; 0.005 was a "be conservative for ODE" choice from `04-RESEARCH.md` Pitfall 1 that was inherited mechanically into Phase 15 L2 without re-validation under the new B-prior regime.**

**Pyro SVI Tips and Tricks (https://pyro.ai/examples/svi_part_iv.html):** "It is generally a good idea to initialize your guide distributions so that they have low variance" — but the same page warns about *overly* small variance leading to "numerically unstable or otherwise undesirable regions." The guidance is to **start small, then increase if optimization fails** — exactly what should happen here.

### Stephan et al. 2008 (REF-002 hemodynamics) — not directly relevant

Hemodynamic priors are fixed in `task_dcm_model`. Not a contributor to the B failure.

### Zeidman et al. 2019 (PMC6711459) — DCM identifiability practice guide

Cited via Phase 16-RESEARCH Section 3.5. **No actionable claim about init_scale,** but reinforces that B identifiability is a known weak point in DCM and posterior diagnostics (shrinkage, coverage) are the canonical checks. RECOV-07 surfacing 0.008 is *exactly* the diagnostic Zeidman recommends to detect identifiability failure or non-converged inference.

## 7. Confidence-Honest Caveats

1. **Cluster log not directly inspected.** The cluster log is on the results branch (`origin/results/phase16-acceptance-20260424-214708`) at `cluster/logs/pytest_phase16_54933838.log`. This research relied on the failure summary in the orchestrator-supplied context. **Recommendation for the planner:** the first plan should include a Task 0 that fetches the log and confirms the per-element B_post means match the ~3% collapse prediction. If the log shows different posterior means, this research's confidence drops to MEDIUM and Experiment 2 (B_true magnitude sanity) becomes more important.

2. **Gradient-flow argument is theoretical, not empirically verified.** I have not directly inspected the Pyro autoguide source for AutoNormal scale-parameter gradient flow. The "stuck at init" interpretation is consistent with the arithmetic and with Pyro documentation, but a direct gradient-norm trace during SVI on a single seed would confirm it definitively. Experiment 1's diagnostics (loss tail, posterior std evolution) will surface this.

3. **The "SPM12-style init = sigma_prior" claim** is paraphrased from `spm_dcm_fmri.m` source (cited in CLAUDE.md as a fallback reference). I have not directly read SPM12 in this research session. The claim is well-established in DCM literature (VL initializes posterior at prior covariance) but if the planner wants to cite it directly, they should pull `spm_dcm_fmri_priors.m` line numbers from a Phase 6 validation reference.

4. **n_eff = 24 vs Rowe threshold 20 is 1.2× margin, not 5×.** Phase 16-RESEARCH Section 3.5 noted this margin. If init_scale fix alone doesn't suffice, the marginal modulator schedule may be a contributing limiter — but addressing it would change the v0.3.1 HGF design and is out of scope.

5. **A 1.7× std growth from init may actually be PHYSICALLY MEANINGFUL.** It's possible the posterior really IS narrow because the data informativeness is high (i.e., the model is correctly very confident). If true, the posterior MEAN should be close to truth and B-RMSE should be small. Since B-RMSE is huge, the posterior mean is wrong — so the "narrow posterior at wrong mean" interpretation (init artifact) is the only one consistent with both signals. This eliminates the alternative reading.

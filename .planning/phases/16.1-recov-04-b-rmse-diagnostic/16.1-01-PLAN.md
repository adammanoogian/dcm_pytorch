  <action>
Create `scripts/diagnose_phase16_init_scale.py` as a single-file, self-contained
diagnostic harness. It is a one-shot script (NOT imported by the package, NOT
imported by tests). The script:

1. Imports the existing helpers verbatim — DO NOT FORK ground-truth construction:
   ```python
   from benchmarks.runners.task_bilinear import (
       _make_bilinear_ground_truth,
       _fit_and_extract,
       _DT_MODEL,
       _DURATION,
       _TR,
   )
   ```
   The script uses `_make_bilinear_ground_truth(3, 42)` to produce the fixture
   and `_fit_and_extract(model_args, model_kwargs, guide_type="auto_normal",
   init_scale=<sweep>, num_steps=500, elbo_type="trace_mean_field")` to fit.
   Construct `model_args` exactly as in `task_bilinear.run_task_bilinear_svi`
   (lines 706-716): `a_mask = ones(N,N)`, `c_mask = zeros(N,1); c_mask[0,0]=1`,
   `t_eval = arange(0, _DURATION, _DT_MODEL, dtype=float64)`. Use
   `bilinear_kwargs = {"b_masks": [data["b_mask_0"]], "stim_mod": data["stim_mod"]}`.

2. Sweeps `init_scale ∈ [0.005, 0.05, 0.1, 0.5]` (in this order so the failing
   baseline runs first and confirms the harness reproduces the cluster signature
   before any "fix" candidate runs).

3. For each init_scale, calls `_fit_and_extract` inside a try/except that
   catches `RuntimeError` (NaN ELBO at step k). Records:
     - `init_scale`
     - `b_rmse_nonnull`: RMSE between `posterior_bi["B_free_0"]["mean"]` (a
       3x3 tensor; mask-weighted) and `data["B_true"][0]` restricted to the
       non-null mask `b_mask_0 > 0` (the two locked elements [1,0]=0.4 and
       [2,1]=0.3). Use `compute_b_rmse_magnitude` from
       `benchmarks.bilinear_metrics` if shape-compatible; else compute inline:
       `sqrt(mean(((B_post * b_mask_0) - B_true[0]) ** 2 [where b_mask_0>0]))`.
     - `a_rmse_bi`: as `compute_rmse(data["A_true"], parameterize_A(posterior_bi["A_free"]["mean"]))`
       (already used in task_bilinear.py line 734).
     - `B_post_mean_per_element`: full 3x3 posterior mean (list of list of float).
     - `B_post_std_per_element`: full 3x3 posterior std (list of list of float).
     - `shrinkage_nonnull_mean`: mean of B_post_std over non-null elements (
       directly from `compute_shrinkage` since `sigma_prior=1.0`).
     - `final_loss_tail`: last 20 ELBO loss values.
     - `nan_at_step`: integer step of NaN, or `null` if completed.
     - `wall_seconds`: float.

4. After all sweeps, picks the WINNER per the criterion:
     - `b_rmse_nonnull < 0.20` AND
     - `shrinkage_nonnull_mean ∈ [0.05, 0.6]` AND
     - `a_rmse_bi <= 0.15` (no catastrophic A regression).
   Tie-break: smallest init_scale that satisfies all three. If no init_scale
   satisfies all three, emit `"chosen_init_scale": null` and a structured
   `escalation` block explaining which gate failed at each candidate.

5. Writes:
     - `.planning/phases/16.1-recov-04-b-rmse-diagnostic/diagnostics/init_scale_sweep_seed42.json`
       (full machine-readable record; created with `Path.mkdir(parents=True, exist_ok=True)`).
     - `.planning/phases/16.1-recov-04-b-rmse-diagnostic/diagnostics/init_scale_sweep_seed42.md`
       (human table with columns init_scale | b_rmse_nonnull | a_rmse_bi |
       shrinkage_nonnull_mean | final_loss_last | nan_at_step | wall_s; followed
       by a "Chosen init_scale: <value>" line and an arithmetic check vs the
       research prediction "0.34 = 96.8% of collapse-to-zero 0.354 at init_scale=0.005").

6. Sets `pyro.clear_param_store()` and seeds (`torch.manual_seed(42)`,
   `pyro.set_rng_seed(42)`) BEFORE each sweep iteration; this is the same
   contract that `_fit_and_extract` uses internally for parameter-store
   isolation.

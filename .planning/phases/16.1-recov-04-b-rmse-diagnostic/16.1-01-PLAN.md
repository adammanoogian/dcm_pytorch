<objective>
Run a deterministic single-seed init_scale sweep on seed 42 against the existing
bilinear ground-truth fixture (`_make_bilinear_ground_truth(3, 42)`), at the
locked Phase 16 acceptance step count (`n_svi_steps = 500`), across init_scale
values `{0.005, 0.05, 0.1, 0.5}`, capturing posterior B mean/std per element,
B-RMSE on the non-null mask, A-RMSE, ELBO tail, and NaN status. Use the
diagnostic to identify the smallest init_scale that pushes B-RMSE below 0.20
AND moves posterior std on non-null elements materially off the init basin
(target: shrinkage_nonnull means in `[0.05, 0.6]`). The winner is consumed
verbatim by Plan 16.1-02.

**Execution layer:** The 4-init sweep runs as a SLURM sbatch job on the Monash
M3 cluster per the user's standing cluster policy (`feedback_cluster_longrun_rule`:
"Submit anything expected >5 min to Monash M3 via sbatch; do not run locally").
Expected wall time: ~30-40 min (4 fits x ~5 min each + harness overhead). A
single LOCAL pre-check (one fit at init_scale=0.005, the failing baseline) runs
BEFORE the cluster submission to confirm the harness reproduces the cluster
B-RMSE 0.34 signature within 5%; this single fit is bounded to <5 min wall
time, qualifying for the cluster-policy carve-out for sub-5-min checks.

Purpose: De-risks the 80-120 min cluster acceptance re-run in Plan 16.1-02 by
empirically confirming the research-predicted root cause (init_scale=0.005 is
0.5% of B prior std; B posterior is stuck at the AutoNormal init basin) and
selecting the cheapest single-knob fix on a ~30-40 min cluster sweep. The
local pre-check guards against wasting cluster time on a divergent harness.

Output: Two diagnostic artifacts (machine + human; on the autopushed results
branch) plus a SUMMARY documenting the chosen `_BILINEAR_INIT_SCALE` value
(or escalation if no init_scale wins, or harness-mismatch escalation if the
local pre-check diverges from the cluster signature).
</objective>

   catches `RuntimeError` (NaN ELBO at step k). Records:
     - `init_scale`
     - `b_rmse_nonnull`: RMSE between `posterior_bi["B_free_0"]["mean"]` (a
       3x3 tensor; mask-weighted) and `data["B_true"][0]` restricted to the
       non-null mask `b_mask_0 > 0` (the two locked elements [1,0]=0.4 and
       [2,1]=0.3). Use `compute_b_rmse_magnitude` from
       `benchmarks.bilinear_metrics` if shape-compatible; else compute inline:
       `sqrt(mean(((B_post * b_mask_0) - B_true[0]) ** 2 [where b_mask_0>0]))`.
     - `a_rmse_bi`: as `compute_rmse(data["A_true"], parameterize_A(posterior_bi["A_free"]["mean"]))`
       (already used in task_bilinear.py line 734).
     - `B_post_mean_per_element`: full 3x3 posterior mean (list of list of float).
     - `B_post_std_per_element`: full 3x3 posterior std (list of list of float).
     - `shrinkage_nonnull_mean`: mean of B_post_std over non-null elements
       (directly from `compute_shrinkage` since `sigma_prior=1.0`).
     - `final_loss_tail`: last 20 ELBO loss values.
     - `nan_at_step`: integer step of NaN, or `null` if completed.
     - `wall_seconds`: float.

4. After all sweeps, picks the WINNER per the criterion:
     - `b_rmse_nonnull < 0.20` AND
     - `shrinkage_nonnull_mean ∈ [0.05, 0.6]` AND
     - `a_rmse_bi <= 0.15` (no catastrophic A regression).
   Tie-break: smallest init_scale that satisfies all three. If no init_scale
   satisfies all three, emit `"chosen_init_scale": null` and a structured
   `escalation` block explaining which gate failed at each candidate.

5. Writes:
     - `--output-json` (full machine-readable record; created with
       `Path(...).parent.mkdir(parents=True, exist_ok=True)`).
     - `--output-md` (human table with columns init_scale | b_rmse_nonnull |
       a_rmse_bi | shrinkage_nonnull_mean | final_loss_last | nan_at_step |
       wall_s; followed by a "Chosen init_scale: <value>" line and an arithmetic
       check vs the research prediction "0.34 = 96.8% of collapse-to-zero
       0.354 at init_scale=0.005").

6. Sets `pyro.clear_param_store()` and seeds (`torch.manual_seed(seed)`,

   contract that `_fit_and_extract` uses internally for parameter-store

  <action>
    Implement the rDCM simulator and update package exports. Use `from __future__ import annotations`, NumPy-style docstrings, type hints.

    **`rdcm_simulator.py`:**

    **`make_stable_A_rdcm(nr, density=0.5, diag_range=(-0.8, -0.2), offdiag_range=(-0.3, 0.3), seed=None) -> tuple[torch.Tensor, torch.Tensor]`:**
    - Generate a stable A matrix suitable for rDCM simulation.
    - density: fraction of off-diagonal connections present.
    - Returns (A, a_mask) where A is the connectivity matrix and a_mask is the binary architecture mask.
    - Diagonal elements: uniform in diag_range.
    - Off-diagonal: first sample binary mask with given density, then fill present connections with uniform in offdiag_range.
    - Verify stability: all eigenvalues of A have negative real parts. If not, scale A until stable.
    - Uses torch.Generator for reproducibility if seed provided.
    - a_mask has 1s on diagonal and where off-diagonal connections exist.

    **`make_block_stimulus_rdcm(n_time, n_inputs, u_dt, block_duration=20.0, rest_duration=20.0, seed=None) -> torch.Tensor`:**
    - Generate block design stimulus for rDCM.
    - Returns u: (n_time, n_inputs) float64 with alternating on/off blocks.
    - Each input has its own block pattern, shifted by a random offset.

    **`simulate_rdcm(A, C, u, u_dt, y_dt, SNR=3.0, a_mask=None, c_mask=None, mode='rigid', sparse_kwargs=None, confound_cols=1, seed=None) -> dict`:**
    - End-to-end rDCM simulation and inversion.
    - Steps:
      1. Generate BOLD: `bold_result = generate_bold(A, C, u, u_dt, y_dt, SNR)`.
      2. Create regressors: `X, Y, N_eff = create_regressors(bold_result['hrf'], bold_result['y'], u, u_dt, y_dt)`.
      3. If a_mask is None, use ones(nr, nr) (full connectivity). If c_mask is None, use ones(nr, nu).
      4. Run inversion:
         - mode='rigid': `result = rigid_inversion(X, Y, a_mask, c_mask, confound_cols)`.
         - mode='sparse': `result = sparse_inversion(X, Y, a_mask, c_mask, confound_cols, **sparse_kwargs)`.
      5. Combine results: bold_result + inversion result + ground truth info.
    - Returns dict with keys:
      - 'A_true', 'C_true': ground truth parameters.
      - 'A_mu', 'C_mu': posterior means.
      - 'y', 'y_clean': noisy and clean BOLD.
      - 'F_total': total free energy.
      - 'F_per_region': per-region free energies.
      - 'iterations_per_region': iteration counts.
      - 'mu_per_region', 'Sigma_per_region': per-region posteriors.
      - 'z_per_region': (sparse only) binary indicators.
      - 'mode': 'rigid' or 'sparse'.
    - Cite: Julia generate_BOLD.jl, rigid_inversion.jl / sparse_inversion.jl.

    - `test_make_block_stimulus_binary`: All values are 0 or 1.
    - `test_make_block_stimulus_has_blocks`: There are transitions between 0 and 1 (not all-zero or all-one).

    **End-to-end rigid recovery (key test):**

    - `test_rigid_recovery_3_region`: (may be slow — use pytest.mark.slow)

      - C = [[1, 0], [0, 1], [0, 0]] (2 inputs to regions 1, 2).
      - Generate block stimulus: n_time=5000, u_dt=0.01, y_dt=2.0.
      - Run simulate_rdcm with mode='rigid', SNR=3.0.
      - Compute Pearson correlation between A_true (flattened) and A_mu (flattened).
      - Assert correlation > 0.8.
      - Verify A_mu diagonal is negative (self-inhibition recovered).
      - Verify free energy is finite.

    - `test_rigid_recovery_diagonal_dominance`: Known A with strong diagonal (-0.5) and weak off-diagonal (0.05). Posterior A_mu diagonal should have larger magnitude than off-diagonal.

    **End-to-end sparse recovery (key test):**

    - `test_sparse_recovery_5_region`: (may be slow — use pytest.mark.slow)
      - Create a 5-region sparse network with density=0.3 using make_stable_A_rdcm(5, density=0.3, seed=42).

      - Generate block stimulus, run simulate_rdcm with mode='sparse', n_reruns=10 (reduced for test speed).
      - Compute F1 score for binary sparsity pattern: true_pattern = (A_true != 0), inferred_pattern = (z > 0.5).
      - Assert F1 > 0.85.
      - Verify absent connections have z < 0.5.
      - Verify present connections have z > 0.5 (most of them).

    - `test_rigid_vs_sparse_fully_connected`: For a fully connected 3-region A, run both rigid and sparse. Both should produce similar A_mu (correlation between them > 0.9). The sparse version's z should be mostly > 0.5 for all true connections.

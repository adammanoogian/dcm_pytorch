  <action>
Create `benchmarks/fixtures.py` with a `load_fixture` function.

```python
from __future__ import annotations

import json
from pathlib import Path

import numpy as np
import torch


def load_fixture(
    variant: str,
    n_regions: int,
    index: int,
    fixtures_dir: str = "benchmarks/fixtures",
) -> dict[str, torch.Tensor]:
    """Load a single .npz fixture as a dict of torch tensors.

    Parameters
    ----------
    variant : str
        One of ``"task"``, ``"spectral"``, ``"rdcm"``.
    n_regions : int
        Number of regions (3, 5, or 10).
    index : int
        Dataset index (0-based).
    fixtures_dir : str
        Root fixtures directory.

    Returns
    -------
    dict[str, torch.Tensor]
        Tensors keyed by field name. Complex arrays (CSD, rDCM
        regressors) are reconstructed from real/imag parts.

    Raises
    ------
    FileNotFoundError
        If the fixture file does not exist.
    """
```

Implementation:
1. Build path: `{fixtures_dir}/{variant}_{n_regions}region/dataset_{index:03d}.npz`
2. Load with `np.load(path)`, convert each array to `torch.from_numpy(arr)`
3. Reconstruct complex tensors: if keys `csd_real` and `csd_imag` exist, create
   `csd = torch.complex(real, imag)` and add to dict (keep real/imag too for
   runners that want them)
4. Same reconstruction for `X_real`/`X_imag` -> `X` and `Y_real`/`Y_imag` -> `Y`
5. Return dict of tensors

Also add a helper `get_fixture_count(variant, n_regions, fixtures_dir)` that
reads the manifest.json or counts .npz files to know how many datasets exist.

The module docstring should describe the fixture directory layout:
```
benchmarks/fixtures/{variant}_{N}region/dataset_{NNN}.npz
```

Do NOT add `from __future__ import annotations` -- wait, DO add it (project convention).

Create `benchmarks/generate_fixtures.py` as a CLI script.

**Structure:**

Three generation functions, one per variant:

`generate_task_fixtures(n_regions, n_datasets, seed, output_dir)`:
- For i in range(n_datasets):
  - seed_i = seed + i  (canonical pattern)
  - torch.manual_seed(seed_i)
  - A_true = make_random_stable_A(n_regions, density=0.5, seed=seed_i)
  - C = zeros(N, 1); C[0,0] = 1.0
  - stim = make_block_stimulus(n_blocks=5, block_duration=15.0, rest_duration=15.0, n_inputs=1)
  - sim = simulate_task_dcm(A_true, C, stim, duration=90.0, dt=0.01, TR=2.0, SNR=5.0, seed=seed_i)
  - Save to `{output_dir}/task_{n_regions}region/dataset_{i:03d}.npz` with fields:
    A_true, C, bold, bold_clean, stimulus (save stim["times"] and stim["values"]),
    TR, SNR, duration, seed
  - Use np.savez; convert tensors via .numpy() or .detach().numpy()
- Write manifest.json: {n_datasets, seed, n_regions, variant, fields list}

`generate_spectral_fixtures(n_regions, n_datasets, seed, output_dir)`:
- For i in range(n_datasets):
  - seed_i = seed + i
  - A_true = make_stable_A_spectral(n_regions, seed=seed_i)
  - sim = simulate_spectral_dcm(A_true, TR=2.0, n_freqs=32, seed=seed_i)
  - Add noise matching spectral_svi.py pattern: decompose, add Gaussian noise at SNR=10
  - Save: A_true, csd_real, csd_imag (split complex CSD), freqs, noisy_csd_real,
    noisy_csd_imag, TR, n_freqs, seed
  - NOTE: Store BOTH clean and noisy CSD so runners can choose

`generate_rdcm_fixtures(n_regions, n_datasets, seed, output_dir)`:
- Match `_generate_rdcm_data` from rdcm_vb.py exactly:
  - seed_i = seed + i
  - A, a_mask = make_stable_A_rdcm(n_regions, density=0.5, seed=seed_i)
  - C = zeros(N, 2); C[0,0]=1; C[1,1]=1 (or C[0,0]=1; C[0,1]=1 if N==1)
  - c_mask = (C != 0).float()
  - u = make_block_stimulus_rdcm(n_time=4000, n_inputs=2, u_dt=0.5, seed=seed_i)
  - torch.manual_seed(seed_i + 10000)
  - bold_result = generate_bold(A, C, u, u_dt=0.5, y_dt=2.0, SNR=3.0)
  - Save: A_true, C_true, a_mask, c_mask, y (bold_result["y"]), y_clean,
    u, u_dt, y_dt, SNR, seed
  - Do NOT store regressors X, Y (runners call create_regressors themselves)

**CLI argparse:**


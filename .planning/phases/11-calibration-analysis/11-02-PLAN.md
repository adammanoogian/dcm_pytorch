2. Add `plot_calibration_curves` function:

```python
def plot_calibration_curves(
    results: dict,
    output_dir: str,
    n_regions: int = 3,
    param_type: str = "all",
    formats: tuple[str, ...] = ("png",),
) -> None:
```

- Groups results by variant (task, spectral, rdcm). NEVER aggregate across variants (P9).
- For each variant, creates one subplot.
- Figure layout: 1xM grid where M = number of variants present in results.
- X-axis: nominal CI levels [0.50, 0.75, 0.90, 0.95]
- Y-axis: observed coverage (0 to 1)
- Plot y=x diagonal as black dashed reference line
- For each guide type present in results at the given n_regions:
  - Extract coverage_multi (or coverage_diag_multi / coverage_offdiag_multi depending on param_type)
  - Compute median coverage at each level across datasets
  - Compute IQR (25th, 75th percentile) as confidence band
  - Plot median as colored line with circle markers, fill_between for IQR band (alpha=0.2)
  - Use GUIDE_COLORS and GUIDE_LABELS for consistent styling
- Filter result keys by suffix `_{n_regions}` to match the requested network size
- Parse guide_type from result key: split by "_" and reconstruct (e.g., "spectral_auto_normal_trace_elbo_3" -> guide_type="auto_normal")
- Title each subplot with variant name and N=n_regions
- Legend outside right or below subplots
- Save via _save_figure with filename "calibration_curves_{param_type}_N{n_regions}"

3. Add `generate_comparison_table` function:

```python
def generate_comparison_table(
    results: dict,
    output_dir: str,
    n_regions: int = 3,
) -> dict[str, str]:
```

- Groups results by variant. Produces one table section per variant.
- For each variant, rows = guide types present. Columns: Method, RMSE, Coverage@90%, Pearson r, Wall Time (s)
- Each cell: "median (q25-q75)" format. Extract from the per-dataset lists (rmse_list, time_list, etc.). For coverage@90%, use coverage_multi["0.9"] or coverage_multi[0.9] list.
- Generate three output formats:
  a. Markdown string -> write to `{output_dir}/comparison_table_N{n_regions}.md`
  b. LaTeX string -> write to `{output_dir}/comparison_table_N{n_regions}.tex`
  c. JSON -> write to `{output_dir}/comparison_table_N{n_regions}.json`
- Return dict with keys "markdown", "latex", "json" containing the string/dict content
- Use np.median, np.percentile for statistics. Never aggregate across variants.

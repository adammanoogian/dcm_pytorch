1. In benchmarks/metrics.py, add a new function `compute_coverage_multi_level`:

```python
def compute_coverage_multi_level(
    true_vals: torch.Tensor,
    samples: torch.Tensor,
    ci_levels: list[float] | None = None,
) -> dict[float, float]:
```

- Default ci_levels = [0.50, 0.75, 0.90, 0.95]
- Use empirical quantiles from samples (NOT z-scores) for accuracy with non-Gaussian posteriors (AutoIAF etc.)
- For each level: alpha = (1.0 - level) / 2.0, compute lo = torch.quantile(samples.float(), alpha, dim=0), hi = torch.quantile(samples.float(), 1-alpha, dim=0), then fraction of true_vals inside [lo, hi]
- Return dict mapping each ci_level float to the coverage float
- NumPy-style docstring citing research Section 3.2

2. Add a helper function `compute_coverage_by_param_type`:

```python
def compute_coverage_by_param_type(
    A_true: torch.Tensor,
    samples: torch.Tensor,
    ci_levels: list[float] | None = None,
) -> dict[str, dict[float, float]]:
```

- Accepts A_true of shape (N,N) and samples of shape (S,N,N)
- Creates diagonal mask and off-diagonal mask
- Calls compute_coverage_multi_level for each subset
- Returns {"all": {...}, "diagonal": {...}, "off_diagonal": {...}}

3. Add `compute_summary_stats` function:

```python
def compute_summary_stats(values: list[float]) -> dict[str, float]:
```

- Returns {"median": ..., "q25": ..., "q75": ..., "mean": ..., "std": ...}
- This standardizes median+IQR reporting (STATE.md risk P12)

def run_calibration_sweep(

**`validation/export_to_mat.py`** -- Three export functions:

1. `export_task_dcm_for_spm(bold_data, stimulus, a_mask, c_mask, TR, u_dt, output_path)`:
   - Build DCM struct dict matching SPM12 `spm_dcm_estimate` requirements exactly (see 06-RESEARCH.md "SPM12 DCM Struct Field Reference")
   - `DCM.Y.y` = bold_data (v, N) float64
   - `DCM.Y.dt` = np.array([[TR]])
   - `DCM.Y.X0` = np.ones((v, 1)) constant confound
   - `DCM.Y.name` = region names as object array
   - `DCM.U.u` = stimulus at MICROTIME resolution (T_micro, M) -- CRITICAL: SPM expects TR/16 resolution
   - `DCM.U.dt` = np.array([[u_dt]])
   - `DCM.U.name` = stimulus names as object array
   - `DCM.a`, `DCM.c` = masks float64
   - `DCM.b` = np.zeros((N, N, 0)) -- no modulatory inputs
   - `DCM.d` = np.zeros((N, N, 0)) -- no nonlinear
   - `DCM.n` = np.array([[N]]), `DCM.v` = np.array([[v]])
   - `DCM.TE` = np.array([[0.04]])
   - `DCM.delays` = np.ones((1, N)) * TR / 2
   - `DCM.options` = dict with nonlinear=0, two_state=0, stochastic=0, centre=0, induced=0, nograph=1, maxit=128
   - ALL scalars wrapped as np.array([[value]]) (2D) per MATLAB convention
   - Save via `scipy.io.savemat(output_path, {'DCM': DCM})`

2. `export_spectral_dcm_for_spm(bold_data, a_mask, c_mask, TR, output_path)`:
   - Same as task DCM struct but with key differences:
   - `DCM.options.induced` = 1 (CSD analysis mode)
   - `DCM.options.analysis` = np.array([['CSD']], dtype=object)
   - `DCM.options.order` = np.array([[8]]) (MAR model order)
   - Export BOLD time series (NOT CSD) -- SPM computes CSD from BOLD internally via MAR model
   - For spectral DCM, stimulus can be minimal/absent: use simple constant input `U.u = np.ones((T_micro, 1))` with `U.dt = TR/16`
   - This ensures fair comparison: both SPM and our code start from the same BOLD

3. `export_rdcm_for_tapas(bold_data, stimulus, a_mask, c_mask, TR, u_dt, output_path)`:
   - Same struct format as task DCM (tapas wraps SPM functions)
   - `DCM.Y.y`, `DCM.U.u`, `DCM.a`, `DCM.c`, `DCM.n`, `DCM.v`, timing fields
   - No special options needed beyond nograph=1

4. Helper `upsample_stimulus(stimulus_tr, TR, microtime_factor=16)`:
   - Upsample from TR resolution to TR/microtime_factor resolution using nearest-neighbor interpolation
   - Pad beginning with 32 zero rows (SPM discards first 32 microtime samples)
   - Return (upsampled_stimulus, u_dt)

**`validation/compare_results.py`** -- Result loading and comparison:

1. `load_spm_results(mat_path)`:
   - Load .mat via `scipy.io.loadmat(mat_path, squeeze_me=False)`
   - Extract `results['Ep_A'][0, 0]` as float64 numpy array
   - Extract `results['F'][0, 0].item()` as float
   - Handle nested struct access carefully (scipy returns structured numpy arrays)
   - Return dict with 'Ep_A', 'F', and any other fields present

2. `load_tapas_results(mat_path)`:
   - Load .mat, extract `results.rigid.Ep`, `results.rigid.logF`, `results.sparse.Ep`, `results.sparse.logF`, `results.sparse.Ip`
   - Return dict with 'rigid' and 'sparse' sub-dicts

3. `compare_posterior_means(pyro_A, ref_A, tolerance=0.10)`:
   - Element-wise comparison using HYBRID metric:
     - For |ref_A[i,j]| > 0.01: use relative error |pyro - ref| / |ref|
     - For |ref_A[i,j]| <= 0.01: use absolute error |pyro - ref| with threshold 0.02
   - Return dict: max_relative_error, mean_relative_error, within_tolerance (bool), element_errors (matrix)

4. `compare_model_ranking(scenarios)`:
   - Each scenario has 'spm_F' (higher=better) and 'pyro_elbo' (higher=better, i.e., negative loss)
   - For all pairs (i, j), check if SPM ranking agrees with Pyro ranking
   - Return agreement_rate (float) and detailed pairwise results
   - Agreement >= 0.80 is the pass criterion

5. `compute_free_param_comparison(pyro_A_free, spm_Ep_A)`:
   - Compare in free parameter space (research finding: SPM Ep.A stores FREE params)
   - Off-diagonal: direct comparison (same space)
   - Diagonal: direct comparison (both are free params; parameterized form = -exp(x)/2)
   - Return same dict structure as compare_posterior_means

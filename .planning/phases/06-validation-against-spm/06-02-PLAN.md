**Update `validation/run_validation.py`** -- Add spectral DCM validation:

`run_spectral_dcm_validation(seed=42, n_regions=3, num_svi_steps=500, n_bold_scans=256, output_dir='validation/data')`:
   - **Step 1: Generate synthetic BOLD** from spectral DCM simulator:
     - Use `simulate_spectral_dcm` to get BOTH the ground truth A and synthetic BOLD time series
     - Use `make_stable_A_spectral(n_regions, density=0.5, seed=seed)`
     - Generate long BOLD time series (n_bold_scans=256 or more) for accurate MAR CSD estimation by SPM
     - TR=2.0, add realistic noise (SNR=5)
   - **Step 2: Export BOLD to .mat** using `export_spectral_dcm_for_spm(bold, a_mask, c_mask, TR, output_path)`
     - CRITICAL: Export BOLD time series, NOT CSD. SPM computes CSD from BOLD internally.
     - This is the apples-to-apples comparison strategy from research
   - **Step 3: Run SPM12 spectral DCM** via subprocess calling `run_spm_spectral_dcm.m`
     - timeout=600
   - **Step 4: Run Pyro SVI** on the same BOLD data:
     - Compute empirical CSD from BOLD using `compute_empirical_csd` (same as Pyro pipeline)
     - Use `spectral_dcm_model` with same A mask
     - Run SVI with 500 steps (calibrated from Phase 5)
     - Extract posterior A_free
   - **Step 5: Load SPM results** and compare A_free vs Ep.A
   - **Step 6: Document CSD estimation difference**:
     - Note that SPM uses MAR model for CSD; our code uses Welch periodogram
     - This is an expected additional source of 5-10% discrepancy beyond VL vs SVI
   - Return dict with comparison results, all parameters, and note about CSD method difference

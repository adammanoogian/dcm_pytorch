**Update `validation/run_validation.py`** -- Add rDCM and model ranking validation:

1. `run_rdcm_validation(seed=42, n_regions=3, n_time=4000, output_dir='validation/data')`:
   - **Step 1: Generate synthetic rDCM data** using `simulate_rdcm`:
     - 3-region network with known A, C matrices
     - n_time=4000 (1000 BOLD scans at TR=2, u_dt=0.5 -- matches Phase 5 rDCM recovery settings)
     - SNR=5
   - **Step 2: Export to .mat** using `export_rdcm_for_tapas`
   - **Step 3: Run tapas rDCM** via subprocess calling `run_tapas_rdcm.m`
     - Handle case where tapas is not installed: return None with message
   - **Step 4: Run our analytic VB** using `rdcm_posterior` rigid inversion (same algorithm family as tapas)
     - Also run sparse VB if tapas sparse results available
   - **Step 5: Load tapas results** using `load_tapas_results`
   - **Step 6: Compare** rigid posterior means and sparse posterior means
   - Return dict with comparison results for both rigid and sparse, or None if tapas unavailable

2. `run_model_ranking_validation(variant='task', seeds=[42, 123, 456], output_dir='validation/data')`:
   - For each seed, create 3 model scenarios with CLEARLY different connectivity:
     - **Model A (correct):** Full 3-region network with true A mask (all connections present)
     - **Model B (missing connection):** Same data, but A mask has one true connection removed
     - **Model C (wrong structure):** Same data, but A mask has a very different pattern (e.g., only diagonal)
   - For each scenario:
     - Run SPM12 estimation with the corresponding mask and extract F (free energy)
     - Run Pyro SVI with the corresponding mask and extract final ELBO (-loss)
   - Compare rankings using `compare_model_ranking`:
     - SPM ranking: higher F = better model
     - Pyro ranking: higher ELBO (lower loss) = better model
     - For correct vs missing-connection, correct should always win
     - For correct vs wrong-structure, correct should always win
   - Return agreement_rate and per-seed results

3. `run_model_ranking_validation_rdcm(seeds=[42, 123, 456])`:
   - Same concept but using rDCM analytic free energy (not SVI ELBO)
   - Compare our analytic F_total with tapas logF rankings
   - If tapas unavailable, compare our rigid vs sparse free energies internally (sanity check that correct model mask wins)

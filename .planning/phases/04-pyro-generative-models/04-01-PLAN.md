    **`task_dcm_model(observed_bold, stimulus, a_mask, c_mask, t_eval, TR, dt=0.5)`:**
    - This is a Pyro model function (not a class). It defines the generative process for task-based DCM.
    - Parameters:
      - `observed_bold`: (T, N) float64 tensor, observed BOLD time series.
      - `stimulus`: PiecewiseConstantInput instance (from ode_integrator).
      - `a_mask`: (N, N) float64 binary tensor, 1 where A connection exists.
      - `c_mask`: (N, M) float64 binary tensor, 1 where C connection exists.
      - `t_eval`: (T_fine,) float64 tensor, fine time grid for ODE integration.
      - `TR`: float, repetition time in seconds.
      - `dt`: float, ODE integration step size (default 0.5 for SVI efficiency; see 04-RESEARCH.md Open Question 4).
    - Implementation:
      1. Extract N from a_mask.shape[0], M from c_mask.shape[1], T from observed_bold.shape[0].
      2. **Sample A_free** from `dist.Normal(zeros(N,N), (1/64)**0.5 * ones(N,N)).to_event(2)`. Apply mask: `A_free = A_free * a_mask`. Apply transform: `A = pyro.deterministic("A", parameterize_A(A_free))`.
      3. **Sample C** from `dist.Normal(zeros(N,M), ones(N,M)).to_event(2)`. Apply mask: `C = C * c_mask`.
      4. **Hemodynamic params: FIXED** -- do NOT sample. Use SPM defaults by passing `hemo_params=None` to CoupledDCMSystem (which uses defaults internally).
      5. **Forward model:** Build `CoupledDCMSystem(A, C, stimulus)`. Create initial state via `make_initial_state(N, dtype=torch.float64)`. Integrate with `integrate_ode(system, y0, t_eval, method="rk4")`. Use "rk4" fixed-step method for predictable runtime in SVI (not "dopri5" which can be slow with stiff systems during optimization).
      6. **Extract BOLD:** From solution (T_fine, 5N), extract log-volume and log-deoxyhemoglobin states: `lnv = solution[:, 3*N:4*N]`, `lnq = solution[:, 4*N:5*N]`. Convert: `v = torch.exp(lnv)`, `q = torch.exp(lnq)`. Compute BOLD: `bold_fine = bold_signal(v, q)`.
      7. **Downsample to TR:** Compute step = round(TR / dt). Subsample: `predicted_bold = bold_fine[::step][:T]`. Register: `pyro.deterministic("predicted_bold", predicted_bold)`.
      8. **Noise precision:** Sample `noise_prec` from `dist.Gamma(tensor(1.0, dtype=float64), tensor(1.0, dtype=float64))`. Compute `noise_std = (1.0 / noise_prec).sqrt()`.
      9. **Likelihood:** Use `pyro.sample("obs", dist.Normal(predicted_bold, noise_std).to_event(2), obs=observed_bold)`. The `.to_event(2)` treats the full (T, N) matrix as a single observation event. This is simpler than nested plates and avoids shape mismatch issues.

    IMPORTANT anti-patterns to avoid:
    - Do NOT sample hemodynamic parameters (kappa, gamma, tau, alpha, E0). They are FIXED.
    - Do NOT use adjoint method (odeint_adjoint) -- use standard odeint for gradient reliability.
    - Do NOT use `pyro.plate` around the ODE integration time steps -- they are NOT independent.
    - Do NOT pass complex tensors to any pyro.sample call.
    - Do NOT use `method="dopri5"` -- use `method="rk4"` for SVI stability. Dopri5 adaptive stepping can cause variable computation graphs across SVI steps.
    - All tensors must be float64. Explicitly pass `dtype=torch.float64` to all tensor constructors.

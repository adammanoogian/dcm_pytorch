    **`rdcm_model(Y, X, a_mask, c_mask, confound_cols=1)`:**
    - Pyro model function wrapping the rDCM frequency-domain regression likelihood.
    - Parameters:
      - `Y`: (N_eff, nr) float64, frequency-domain data from create_regressors.
      - `X`: (N_eff, D) float64, design matrix from create_regressors. D = nr + nu + nc.
      - `a_mask`: (nr, nr) float64 binary mask.
      - `c_mask`: (nr, nu) float64 binary mask.
      - `confound_cols`: int, number of confound columns (default 1).
    - Implementation:
      1. Extract nr from a_mask.shape[0], nu from c_mask.shape[1], nc = confound_cols.
      2. **Region-wise parameter sampling using a Python loop** (NOT a plate for parameters, because each region can have a different number of active connections).
      3. For each region r (0 to nr-1):
         a. Build the active column indices for this region from a_mask and c_mask:
            - A columns: indices where `a_mask[r, :]` is 1 (which columns of X[:, :nr] to select).
            - C columns: indices where `c_mask[r, :]` is 1 (which columns of X[:, nr:nr+nu] to select).
            - Confound columns: always all nc columns (X[:, nr+nu:nr+nu+nc]).
            - D_r = number of active A connections + active C connections + nc.
         b. Select the relevant columns from X to get X_r (N_eff, D_r).
         c. Filter NaN rows: `valid = ~torch.isnan(Y[:, r])`. Apply to both Y_r and X_r.
         d. Sample theta_r: `pyro.sample(f"theta_{r}", dist.Normal(zeros(D_r, dtype=float64), ones(D_r, dtype=float64)).to_event(1))`.
            - Prior: N(0, 1) for all parameters. This is a simple weakly informative prior suitable for the Pyro wrapper. The tight SPM-style priors are already handled by the analytic VB.
         e. Sample noise precision: `pyro.sample(f"noise_prec_{r}", dist.Gamma(tensor(2.0, dtype=float64), tensor(1.0, dtype=float64)))`.
         f. Compute predicted Y_r: `pred_r = X_r_valid @ theta_r`.
         g. Compute noise_std: `(1.0 / noise_prec_r).sqrt()`.
         h. Likelihood: `pyro.sample(f"obs_{r}", dist.Normal(pred_r, noise_std).to_event(1), obs=Y_r_valid)`.
      4. Note: This model is for ELBO comparison and amortization in Phase 7. For routine rDCM inference, users should call `rigid_inversion` or `sparse_inversion` directly. Add a docstring note saying this.

    IMPORTANT notes:
    - Use a Python loop over regions, NOT a pyro.plate for parameters. Each region has a different D_r (number of active connections), so vectorization is not possible.
    - Use `pyro.plate` ONLY for the frequency-domain data points within each region (via `.to_event(1)` on the likelihood, which is equivalent).
    - Prior on theta is N(0, 1) -- intentionally broader than the analytic VB priors, since the Pyro model serves a different purpose (ELBO comparison, amortization).
    - The Y and X inputs come pre-computed from `create_regressors`. This model does NOT call create_regressors internally.

    **`create_guide(model, init_scale=0.01)`:**
    - Factory function that creates an AutoNormal guide for any Pyro model function.
    - Parameters:
      - `model`: callable, the Pyro model function.
      - `init_scale`: float, initial scale for guide distributions (default 0.01 to prevent NaN; see 04-RESEARCH.md Pitfall 1).
    - Returns: `AutoNormal(model, init_scale=init_scale)`.
    - Docstring: Explain that AutoNormal creates a mean-field Gaussian (diagonal Normal) guide with independent loc/scale parameters per latent variable. The init_scale=0.01 starts guide distributions tight to prevent ODE blow-up from extreme initial A_free samples.

    **`run_svi(model, guide, model_args, num_steps=2000, lr=0.01, clip_norm=10.0, lr_decay_factor=0.01, num_particles=1)`:**
    - Run SVI optimization for any Pyro model/guide pair.
    - Parameters:
      - `model`: callable, Pyro model function.
      - `guide`: callable, Pyro guide function (from create_guide or custom).
      - `model_args`: tuple, positional arguments to pass to model and guide.
      - `num_steps`: int, number of SVI steps (default 2000).
      - `lr`: float, initial learning rate (default 0.01).
      - `clip_norm`: float, gradient clipping norm (default 10.0).
      - `lr_decay_factor`: float, decay lr to this fraction of initial (default 0.01 = decay to 1%).
      - `num_particles`: int, number of ELBO particles (default 1).
    - Implementation:
      1. `pyro.clear_param_store()` -- always clear before a new run.
      2. Compute `lrd = lr_decay_factor ** (1.0 / max(num_steps, 1))` for per-step multiplicative LR decay.
      3. Create optimizer: `ClippedAdam({"lr": lr, "clip_norm": clip_norm, "lrd": lrd})`.
      4. Create loss: `Trace_ELBO(num_particles=num_particles, vectorize_particles=(num_particles > 1))`.
      5. Create SVI: `SVI(model, guide, optimizer, loss=elbo)`.
      6. Training loop: for step in range(num_steps), call `loss = svi.step(*model_args)`, append to losses list. If loss is NaN, raise `RuntimeError(f"NaN ELBO at step {step}")`.
      7. Return dict: `{"losses": losses, "final_loss": losses[-1], "num_steps": num_steps}`.

    **`extract_posterior_params(guide, model_args)`:**
    - Extract posterior parameters from a trained AutoNormal guide.
    - Parameters:
      - `guide`: AutoNormal guide instance.
      - `model_args`: tuple, arguments to the model (needed for guide.median()).
    - Implementation:
      1. Get median values: `median = guide.median(*model_args)`.
      2. Get parameter store: `param_store = pyro.get_param_store()`.
      3. Collect all guide parameters (locs and scales) into a dict.
      4. Return dict with keys "median" (from guide.median) and "params" (from param store).
    - This helper simplifies posterior extraction for downstream analysis.

  <name>Task 1: Task DCM parameter recovery test suite</name>
  <files>tests/test_task_dcm_recovery.py</files>
  <action>
Create `tests/test_task_dcm_recovery.py` implementing the task-DCM parameter recovery protocol.

**Shared helpers** (module-level functions):

1. `_pearson_corr(x, y)` -- Pearson correlation between two 1D tensors (copy pattern from `test_rdcm_simulator.py`, avoids numpy process abort on Windows).

2. `compute_rmse_A(A_true, A_inferred)` -- `torch.sqrt(torch.mean((A_true - A_inferred)**2)).item()`.

3. `run_single_task_recovery(seed, n_regions=3, num_steps=3000)` -- Single recovery trial:
   - Generate A via `make_random_stable_A(n_regions, density=0.5, seed=seed)`
   - C: first column of ones for region 0, zeros elsewhere. Shape `(n_regions, 1)`.
   - Stimulus: `make_block_stimulus(n_blocks=5, block_duration=15.0, rest_duration=15.0, n_inputs=1)`
   - Simulate: `simulate_task_dcm(A, C, stim, duration=300.0, dt=0.01, TR=2.0, SNR=5.0, seed=seed)`
   - Build model args: `a_mask = torch.ones(N, N)`, `c_mask` matching C nonzero pattern, `t_eval = torch.arange(0, 300.0, 0.5)`, TR=2.0, dt=0.5
   - Run SVI: `create_guide(task_dcm_model, init_scale=0.01)`, `run_svi(model, guide, model_args, num_steps=num_steps, lr=0.005, clip_norm=10.0, lr_decay_factor=0.01)`
   - Extract posterior: `extract_posterior_params(guide, model_args)`
   - Get A_inferred: `parameterize_A(posterior["median"]["A_free"])`
   - Get 95% CI: `guide.quantiles([0.025, 0.975], *model_args)` for A_free bounds. For **off-diagonal** elements, CI bounds pass through directly. For **diagonal** elements, the transform `a_ii = -exp(A_free_ii)/2` is monotone decreasing, so flip: `A_lo_diag = -exp(A_free_hi_diag)/2`, `A_hi_diag = -exp(A_free_lo_diag)/2`.
   - Return dict with: `A_true`, `A_inferred`, `A_lo`, `A_hi`, `losses`, `final_loss`
   - Wrap the entire trial in try/except RuntimeError to catch NaN ELBO. If caught, return None (skip this seed).

4. `compute_coverage(results_list)` -- For each result, check element-wise `(A_true >= A_lo) & (A_true <= A_hi)`. Return fraction covered.

**Test class `TestTaskDCMRecovery`** (CI-fast, no markers):

- `test_task_dcm_rmse_below_threshold`: Run 10 datasets (seeds 100-109), 3 regions, 3000 steps. Compute mean RMSE across successful runs. Assert mean RMSE < 0.05. Print individual and mean RMSE.
- `test_task_dcm_coverage_calibrated`: Using same 10-dataset results (or re-run), compute coverage. Assert coverage in [0.90, 0.99]. If coverage is below 0.90, print diagnostic (how many missed, which elements).
- `test_task_dcm_correlation_high`: Flatten all (A_true, A_inferred) pairs, compute overall Pearson correlation. Assert > 0.85.
- `test_task_dcm_loss_decreases`: For each run, assert mean of last 20% losses < mean of first 20% losses. Count how many converged; assert >= 8 of 10.

Use a **module-level cache** (global list) to avoid re-running SVI 4 times: run all 10 datasets once in the first test, store results in a module-level `_TASK_RESULTS` list, reuse in subsequent tests. Use `pytest` fixture with `scope="module"` for this.

**Test class `TestTaskDCMRecoverySlow`** (marked `@pytest.mark.slow`):

- `test_task_dcm_recovery_50_datasets_3_regions`: 50 datasets (seeds 200-249), 3 regions, 5000 steps. Assert mean RMSE < 0.05, coverage in [0.90, 0.99], correlation > 0.85.
- `test_task_dcm_recovery_5_regions`: 10 datasets (seeds 300-309), 5 regions, 5000 steps. Same thresholds.

**Important details:**
- All tensors dtype `torch.float64`.
- Use `pyro.set_rng_seed(seed)` AND `torch.manual_seed(seed)` before each SVI run for reproducibility.
- The `c_mask` should match the actual nonzero pattern of C (only region 0 receives input).
- Do NOT use `numpy.corrcoef` (Windows process abort issue).

<task type="auto">
  <name>Task 2: Spectral DCM parameter recovery test suite</name>
  <files>tests/test_spectral_dcm_recovery.py</files>
  <action>
Create `tests/test_spectral_dcm_recovery.py` implementing the spectral DCM parameter recovery protocol.

**Shared helpers** (same patterns as Task 1):

1. `_pearson_corr(x, y)`, `compute_rmse_A(A_true, A_inferred)` -- same as Task 1.

2. `run_single_spectral_recovery(seed, n_regions=3, num_steps=2000)` -- Single recovery trial:
   - Generate A via `make_stable_A_spectral(n_regions, seed=seed)`
   - Simulate: `simulate_spectral_dcm(A, TR=2.0, n_freqs=32, seed=seed)`
   - Add noise to simulated CSD: after simulation, add Gaussian noise to the decomposed real/imaginary CSD vector at SNR=5.0. Compute `signal_power = obs_real.pow(2).mean().sqrt()`, then `noise_std = signal_power / 5.0`, then `noisy_obs = obs_real + noise_std * torch.randn_like(obs_real)`. Reconstruct complex CSD from the noisy real vector. This makes recovery realistic (not trivial).
   - Build model args: `a_mask = torch.ones(N, N)`, pass `(noisy_csd, freqs, a_mask, N)`
   - Run SVI: `create_guide(spectral_dcm_model, init_scale=0.01)`, `run_svi(model, guide, model_args, num_steps=num_steps, lr=0.01, clip_norm=10.0, lr_decay_factor=0.01)`
   - Extract A_inferred via `parameterize_A(posterior["median"]["A_free"])`
   - Get 95% CI via `guide.quantiles([0.025, 0.975], *model_args)` for A_free. Apply same diagonal flip logic as task DCM.
   - Return dict or None on failure.

**Test class `TestSpectralDCMRecovery`** (CI-fast):

- `test_spectral_dcm_rmse_below_threshold`: 10 datasets (seeds 100-109), 3 regions, 2000 steps. Mean RMSE < 0.05.
- `test_spectral_dcm_coverage_calibrated`: Coverage in [0.90, 0.99].
- `test_spectral_dcm_correlation_high`: Correlation > 0.85.
- `test_spectral_dcm_loss_decreases`: At least 8/10 converge (last 20% loss < first 20% loss).

Use module-level cache via `scope="module"` fixture, same pattern as Task 1.

**Test class `TestSpectralDCMRecoverySlow`** (marked `@pytest.mark.slow`):

- `test_spectral_dcm_recovery_50_datasets`: 50 datasets, seeds 200-249, 3000 steps.
- `test_spectral_dcm_recovery_5_regions`: 10 datasets, seeds 300-309, 5 regions, 3000 steps.

**Important details:**
- `simulate_spectral_dcm` returns `{"csd": ..., "freqs": ..., "A": ...}`. The csd is complex128 shape (F, N, N).
- The noise must be added in the real/imaginary decomposed space (matching the model's likelihood), then reassembled into complex CSD for the model_args.
- Use `decompose_csd_for_likelihood` from `spectral_dcm_model.py` for consistent decomposition.
- `pyro.set_rng_seed(seed)` and `torch.manual_seed(seed)` before each SVI run.

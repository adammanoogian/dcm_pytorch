  <action>
Each runner follows the same interface: `def run_VARIANT_METHOD(config: BenchmarkConfig) -> dict`.
Returns a dict with keys: rmse_list, coverage_list, correlation_list, elbo_list, time_list,
n_steps_list, summary (mean/std of each metric), metadata.

All runners MUST:
- Fix seeds: torch.manual_seed, pyro.set_rng_seed, numpy.random.seed
- Use config.n_datasets, config.n_svi_steps, config.n_regions, config.seed
- Time each inference run with time.time()
- Handle failures gracefully (skip failed datasets, require >= 50% success)
- Print progress: "Running dataset {i+1}/{n_datasets}..."
- Return summary statistics: mean +/- std for each metric

1. **benchmarks/runners/task_svi.py** -- `run_task_svi(config)`:

   Reuse patterns from test_task_dcm_recovery.py `run_single_task_recovery`:
   - Generate A via make_random_stable_A (seed=config.seed + i)
   - C: region 0 receives input (C[0,0]=1.0)
   - Stimulus: make_block_stimulus(n_blocks=3 if quick else 5)
   - Duration: 30.0 if quick else 90.0
   - Simulate BOLD via simulate_task_dcm(SNR=5.0)
   - Run SVI: create_guide(task_dcm_model, init_scale=0.01), run_svi with
     lr=0.005, clip_norm=10.0, lr_decay_factor=0.01
   - Extract posterior via extract_posterior_params, parameterize_A
   - Compute RMSE, coverage (95% CI via quantiles), correlation via benchmarks.metrics
   - Record wall time, final ELBO, number of steps

2. **benchmarks/runners/spectral_svi.py** -- `run_spectral_svi(config)`:

   Reuse patterns from test_spectral_dcm_recovery.py `run_single_spectral_recovery`:
   - Generate A via make_stable_A_spectral (seed=config.seed + i)
   - Simulate CSD via simulate_spectral_dcm(TR=2.0, n_freqs=32)
   - Add SNR=10 noise to decomposed real/imag CSD
   - Run SVI: create_guide, run_svi with lr=0.01, lr_decay_factor=0.1
   - n_svi_steps: 500 (spectral converges fast; override config if needed)
   - Extract posterior, compute metrics

3. **benchmarks/runners/rdcm_vb.py** -- `run_rdcm_vb(config)`:

   Reuse patterns from tests/test_rdcm_recovery.py:
   - Generate data via simulate_rdcm (n_regions=3, n_time=4000, u_dt=0.5)
   - Run rigid_inversion from rdcm_posterior
   - Run sparse_inversion from rdcm_posterior
   - Compute RMSE, coverage (from analytic posterior Sigma), correlation
   - For sparse: also compute F1 score (precision/recall of nonzero A elements)
   - Record free energy (not ELBO -- analytic)
   - Return separate entries for "rdcm_rigid" and "rdcm_sparse"

Coding conventions: `from __future__ import annotations`, NumPy docstrings,
ruff line-length 88. Import metrics from benchmarks.metrics. Import config from
benchmarks.config.

   - In quick mode, train a CI-scale guide inline: 50 datasets, 100 SVI steps
     (matching test_amortized_benchmark.py pattern). This produces rough but
     functional metrics.
   - For each test dataset:
     a. Generate data (same seed protocol as task_svi runner)
     b. Forward pass through guide: guide.sample_posterior(bold, n_samples=500)
     c. Compute RMSE, coverage from samples (90% CI, 1.645 z-score)
     d. Time the inference (forward pass only, not training)
   - Also run per-subject SVI on same data (to compute amortization gap)
   - Compute amortization gap via benchmarks.metrics.compute_amortization_gap
   - Return: rmse_list, coverage_list, rmse_ratio_list, speed_ratio_list,
     amortization_gap_list, summary

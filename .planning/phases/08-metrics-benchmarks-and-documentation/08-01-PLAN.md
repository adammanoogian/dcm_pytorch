Create the benchmarks/ package structure:

1. **benchmarks/__init__.py**: Package init, empty or with brief docstring.

2. **benchmarks/metrics.py**: Consolidate these identical helpers that appear in
   test_task_dcm_recovery.py, test_spectral_dcm_recovery.py, and
   test_amortized_benchmark.py:

   - `compute_rmse(A_true: torch.Tensor, A_inferred: torch.Tensor) -> float`
     RMSE between two tensors. Same as existing `compute_rmse_A`.

   - `pearson_corr(x: torch.Tensor, y: torch.Tensor) -> float`
     Manual Pearson correlation (no numpy.corrcoef -- Windows abort issue,
     see STATE.md decision). Same as existing `_pearson_corr`.

   - `compute_coverage_from_ci(A_true: torch.Tensor, A_lo: torch.Tensor, A_hi: torch.Tensor) -> float`
     Fraction of elements where A_true falls in [A_lo, A_hi].
     Extracted from existing `compute_coverage` in recovery test files.

   - `compute_coverage_from_samples(true_vals: torch.Tensor, samples: torch.Tensor, ci_level: float = 0.90) -> float`
     Coverage from posterior samples using z-score CI (1.645 for 90%).
     Extracted from test_amortized_benchmark.py coverage computation.

   - `compute_amortization_gap(elbo_svi: float, elbo_amortized: float) -> dict`
     Returns {"absolute_gap": ..., "relative_gap": ...}.
     relative_gap = gap / |elbo_svi|.

   All functions: `from __future__ import annotations`, type hints, NumPy docstrings,
   torch-only (no numpy dependency except for return types).

3. **benchmarks/config.py**: BenchmarkConfig dataclass:

   ```python
   @dataclass
   class BenchmarkConfig:
       variant: str               # "task", "spectral", "rdcm"
       method: str                # "svi", "amortized", "vb"
       n_datasets: int = 20
       n_regions: int = 3
       n_svi_steps: int = 3000
       seed: int = 42
       quick: bool = False
       output_dir: str = "benchmarks/results"
       save_figures: bool = True
       figure_dir: str = "figures"
   ```

   Add a `quick_config()` classmethod that returns a config with reduced params:
   - task: n_datasets=3, n_svi_steps=500
   - spectral: n_datasets=5, n_svi_steps=500
   - rdcm: n_datasets=5

   Add a `full_config()` classmethod for paper-quality runs:
   - task: n_datasets=20, n_svi_steps=3000
   - spectral: n_datasets=50, n_svi_steps=500
   - rdcm: n_datasets=50

4. **benchmarks/runners/__init__.py**: Runner registry.
   Define RUNNER_REGISTRY as a dict mapping (variant, method) tuples to runner
   functions. Initially populate with placeholder entries that raise NotImplementedError

   ```python
       ("task", "svi"): _not_implemented,

       ("spectral", "svi"): _not_implemented,
       ("spectral", "amortized"): _not_implemented,
       ("rdcm", "vb"): _not_implemented,
       ("spm", "reference"): _not_implemented,
   }
   ```

   Import Callable from collections.abc. The runners will be registered in 08-02.

Coding conventions: `from __future__ import annotations` in every file, ruff line-length 88,
NumPy-style docstrings, Python 3.11+ type hints.

1. **benchmarks/run_all_benchmarks.py**: CLI entry point using argparse (matching
   existing pattern from scripts/generate_training_data.py and
   scripts/train_amortized_guide.py).

   Arguments:
   - `--variant`: choices=["task", "spectral", "rdcm", "all"], default="all"
   - `--method`: choices=["svi", "amortized", "vb", "all"], default="all"
   - `--quick`: store_true, "Use reduced dataset counts and SVI steps for CI/dev"
   - `--seed`: int, default=42
   - `--output-dir`: str, default="benchmarks/results"
   - `--no-figures`: store_true, "Skip figure generation"

   Main logic:
   a. Parse args.
   b. Determine which (variant, method) pairs to run based on --variant/--method.
      Valid combinations: (task, svi), (task, amortized), (spectral, svi),
      (spectral, amortized), (rdcm, vb), (spm, reference).
      Skip invalid combos silently (e.g., (rdcm, svi) is not valid).
   c. For each valid combo, look up RUNNER_REGISTRY. If runner raises
      NotImplementedError, print warning and skip. If runner is registered,
      create BenchmarkConfig (using quick_config if --quick, else full_config)
      and call runner(config).
   d. Collect all results into a master dict.
   e. Add metadata: timestamp (ISO 8601), git commit hash (via subprocess),
      Python version, PyTorch version, platform info.
   f. Save to `{output_dir}/benchmark_results.json` using json.dump with
      indent=2. Convert torch tensors to lists before serialization.
   g. Print summary table to stdout using simple string formatting (no tabulate
      required in core path -- use tabulate if available, fall back to manual
      formatting).

   Include `if __name__ == "__main__":` guard.

   The script should handle MATLAB not being available gracefully: if the SPM
   reference runner fails to load results, print "SPM12 results not available
   (MATLAB required)" and continue.

1. Add import for Predictive:

2. Replace the entire `extract_posterior_params` function with a Predictive-based
   implementation. New signature:
   ```python
   def extract_posterior_params(
       guide: AutoGuide,
       model_args: tuple[Any, ...],
       model: Callable[..., Any] | None = None,
       num_samples: int = 1000,
   ) -> dict[str, Any]:
   ```
   The `model` parameter is needed by Predictive. If None, attempt to get it from
   `guide.model` (all AutoGuide subclasses store the model). This keeps backward
   compatibility -- existing callers don't pass model.

3. Implementation:
   ```python
   if model is None:
       model = guide.model

   predictive = Predictive(
       model,
       guide=guide,
       num_samples=num_samples,
       return_sites=None,  # Return all latent sites
   )
   
   with torch.no_grad():
       samples = predictive(*model_args)
   
   result: dict[str, Any] = {}
   for site_name, tensor in samples.items():
       # Skip observed sites (contain _obs suffix pattern)
       # or sites that are deterministic
       result[site_name] = {
           "mean": tensor.float().mean(dim=0),
           "std": tensor.float().std(dim=0),
           "samples": tensor,
       }
   
   # Backward compatibility: provide "median" key mapping site names
   # to mean values (approximation of median for symmetric distributions)
   result["median"] = {
       name: info["mean"] for name, info in result.items()
       if isinstance(info, dict) and "mean" in info
   }
   
   return result
   ```

   IMPORTANT: The "median" key at the top level preserves backward compatibility
   with existing code that does `posterior["median"]["A_free"]`. The new per-site
   dicts with mean/std/samples are ALSO at the top level.

4. Update the docstring fully:
   - Document the new sample-based approach
   - Document model parameter (optional, extracted from guide if not provided)
   - Document num_samples parameter
   - Document return dict structure (per-site mean/std/samples + "median" compat key)
   - Note that for AutoDelta, all samples are identical and std=0

5. Update type annotation: guide parameter from `AutoNormal` to `AutoGuide`.

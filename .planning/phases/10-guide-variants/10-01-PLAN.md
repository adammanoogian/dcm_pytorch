<task type="auto">
  <name>Task 1: Extend create_guide factory with 6 guide types and blocklist</name>
  <files>src/pyro_dcm/models/guides.py</files>
  <action>
Modify `create_guide` in `src/pyro_dcm/models/guides.py`:

1. Add imports at the top of the file:
   ```python
   from pyro.infer.autoguide import (
       AutoDelta,
       AutoGuide,
       AutoIAFNormal,
       AutoLaplaceApproximation,
       AutoLowRankMultivariateNormal,
       AutoMultivariateNormal,
       AutoNormal,
   )
   ```
   Remove the existing standalone `from pyro.infer.autoguide import AutoNormal`.

2. Create a module-level registry dict mapping string keys to Pyro classes:
   ```python
   GUIDE_REGISTRY: dict[str, type[AutoGuide]] = {
       "auto_delta": AutoDelta,
       "auto_normal": AutoNormal,
       "auto_lowrank_mvn": AutoLowRankMultivariateNormal,
       "auto_mvn": AutoMultivariateNormal,
       "auto_iaf": AutoIAFNormal,
       "auto_laplace": AutoLaplaceApproximation,
   }
   ```

3. Define which guides accept init_scale:
   ```python
   _INIT_SCALE_GUIDES: set[str] = {"auto_normal", "auto_lowrank_mvn", "auto_mvn"}
   ```

4. Define mean-field guide set (needed by Plan 10-02, define here so it's co-located):
   ```python
   MEAN_FIELD_GUIDES: set[str] = {"auto_delta", "auto_normal"}
   ```

5. Define the blocklist as a module-level dict:
   ```python
   _MAX_REGIONS: dict[str, int] = {
       "auto_mvn": 7,
   }
   ```
   (Block auto_mvn at N >= 8, i.e., max allowed is 7)

6. Change the `create_guide` signature to:
   ```python
   def create_guide(
       model: Callable[..., Any],
       *,
       guide_type: str = "auto_normal",
       init_scale: float = 0.01,
       n_regions: int | None = None,
       **kwargs: Any,
   ) -> AutoGuide:
   ```
   IMPORTANT: `guide_type` and all new parameters are keyword-only (after `*`).
   The `model` positional arg stays. Existing calls like `create_guide(model, init_scale=0.01)`
   still work because `init_scale` was already passed as a keyword in ALL existing call sites
   (verified via grep -- every call uses `init_scale=0.01` as a named kwarg).

7. Inside create_guide, implement:
   a. Validate guide_type is in GUIDE_REGISTRY, raise ValueError with available keys if not.
   b. Check blocklist: if n_regions is not None and guide_type has a max in _MAX_REGIONS,
      raise ValueError with suggestion to use 'auto_lowrank_mvn' instead.
   c. Build constructor kwargs based on guide_type:
      - If guide_type in _INIT_SCALE_GUIDES: pass init_scale
      - If guide_type == "auto_lowrank_mvn": also pass rank=kwargs.pop("rank", 2)
      - If guide_type == "auto_iaf": pass num_transforms=kwargs.pop("num_transforms", 2),
        hidden_dim=kwargs.pop("hidden_dim", 20). Do NOT pass init_scale.
      - If guide_type in ("auto_delta", "auto_laplace"): pass NO init_scale.
      - Pass any remaining **kwargs through to the constructor.
   d. Return GUIDE_REGISTRY[guide_type](model, **constructor_kwargs)

8. Update the docstring to document the new parameters (guide_type, n_regions, **kwargs),
   the 6 supported guide types, and the blocklist behavior. Update the return type
   annotation from AutoNormal to AutoGuide.

9. Update the module docstring at the top to reflect the expanded guide factory scope
   (no longer just AutoNormal).

<task type="auto">
  <name>Task 2: Add unit tests for guide factory</name>
  <files>tests/test_guide_factory.py</files>
  <action>
Create `tests/test_guide_factory.py` with the following test structure. Use a minimal
Pyro model for testing (3 latent sites: a ~ Normal(0,1), b ~ Normal(0,1),
obs ~ Normal(a+b, 1)):

1. **Parametrized instantiation test** (`test_create_guide_returns_correct_type`):
   Use `@pytest.mark.parametrize` over all 6 guide_type strings.
   For each: call `create_guide(model, guide_type=X)`, assert the return type
   matches the expected Pyro class (AutoDelta, AutoNormal, etc.).

2. **Default backward compatibility** (`test_create_guide_default_is_auto_normal`):
   Call `create_guide(model)` with no guide_type -- assert returns AutoNormal.
   Call `create_guide(model, init_scale=0.01)` -- assert returns AutoNormal.

3. **init_scale asymmetry** (`test_init_scale_not_passed_to_unsupported_guides`):
   Verify that `create_guide(model, guide_type='auto_delta', init_scale=0.01)`
   does NOT raise (init_scale is silently ignored for unsupported guides).
   Same for 'auto_laplace' and 'auto_iaf'.

4. **Blocklist enforcement** (`test_auto_mvn_blocked_at_high_n_regions`):
   `create_guide(model, guide_type='auto_mvn', n_regions=8)` raises ValueError.
   `create_guide(model, guide_type='auto_mvn', n_regions=7)` does NOT raise.
   Error message contains "auto_lowrank_mvn" as suggestion.

5. **Invalid guide_type** (`test_invalid_guide_type_raises_value_error`):
   `create_guide(model, guide_type='nonexistent')` raises ValueError.
   Error message contains list of valid guide types.

6. **kwargs passthrough** (`test_kwargs_passthrough`):
   `create_guide(model, guide_type='auto_lowrank_mvn', rank=3)` -- verify guide
   is created without error (rank=3 overrides default of 2).
   `create_guide(model, guide_type='auto_iaf', hidden_dim=32, num_transforms=3)` --
   verify guide is created without error.

7. **SVI smoke test per guide type** (`test_svi_step_with_each_guide_type`):
   Parametrize over all 6 guide_type strings. For each:
   - Create guide via create_guide
   - Create SVI with Trace_ELBO
   - Run 1 step of svi.step() -- assert no error and loss is a finite float.
   Mark this test with `@pytest.mark.slow` if it takes > 5s total.

All tests use `from __future__ import annotations`, NumPy-style docstrings on
the test module, and follow project conventions (ruff, 88-char lines).

  <name>Task 1: Add ELBO plumbing to run_svi with AutoLaplace handling</name>
  <files>src/pyro_dcm/models/guides.py, benchmarks/config.py</files>
  <action>
**In `src/pyro_dcm/models/guides.py`:**

1. Add imports for ELBO classes:
   ```python
   from pyro.infer import SVI, Trace_ELBO, TraceMeanField_ELBO, RenyiELBO
   ```
   Remove the existing standalone `from pyro.infer import SVI, Trace_ELBO`.

2. Create an ELBO registry dict:
   ```python
   ELBO_REGISTRY: dict[str, type] = {
       "trace_elbo": Trace_ELBO,
       "tracemeanfield_elbo": TraceMeanField_ELBO,
       "renyi_elbo": RenyiELBO,
   }
   ```

3. Add `elbo_type` parameter to `run_svi`:
   ```python
   def run_svi(
       model: Callable[..., Any],
       guide: Callable[..., Any],
       model_args: tuple[Any, ...],
       num_steps: int = 2000,
       lr: float = 0.01,
       clip_norm: float = 10.0,
       lr_decay_factor: float = 0.01,
       num_particles: int = 1,
       elbo_type: str = "trace_elbo",
       guide_type: str | None = None,
   ) -> dict[str, Any]:
   ```
   `guide_type` is optional -- passed so run_svi can validate ELBO compatibility.
   If None, skip compatibility check (backward-compatible for existing callers
   who don't pass it).

4. Inside run_svi, BEFORE creating the ELBO object:
   a. Validate elbo_type is in ELBO_REGISTRY, raise ValueError if not.
   b. If `elbo_type == "tracemeanfield_elbo"` and `guide_type is not None`
      and `guide_type not in MEAN_FIELD_GUIDES`:
      raise ValueError with message:
      `f"TraceMeanField_ELBO requires a mean-field guide (auto_delta or auto_normal), got '{guide_type}'. Use 'trace_elbo' or 'renyi_elbo' instead."`

5. Create ELBO object based on elbo_type:
   - For "trace_elbo": `Trace_ELBO(num_particles=num_particles, vectorize_particles=(num_particles > 1))`
   - For "tracemeanfield_elbo": `TraceMeanField_ELBO(num_particles=num_particles, vectorize_particles=(num_particles > 1))`
   - For "renyi_elbo": 
     ```python
     renyi_particles = max(num_particles, 2)  # RenyiELBO needs >= 2
     RenyiELBO(alpha=0.5, num_particles=renyi_particles, vectorize_particles=(renyi_particles > 1))
     ```

6. After the SVI loop, handle AutoLaplaceApproximation post-processing:
   ```python
   # Post-processing for AutoLaplaceApproximation
   post_guide = None
   if guide_type == "auto_laplace":
       post_guide = guide.laplace_approximation(*model_args)
   ```

7. Update the return dict to include the post-Laplace guide:
   ```python
   result = {
       "losses": losses,
       "final_loss": losses[-1],
       "num_steps": num_steps,
   }
   if post_guide is not None:
       result["guide"] = post_guide
   return result
   ```

8. Update the run_svi docstring to document elbo_type, guide_type parameters,
   the TraceMeanField_ELBO compatibility check, RenyiELBO alpha=0.5 default,
   and the AutoLaplace post-processing that returns a "guide" key.

**In `benchmarks/config.py`:**

9. Update `guide_type` field default from `"mean_field"` to `"auto_normal"` to
   align with the new GUIDE_REGISTRY string keys. Update the docstring to list
   the 6 valid guide_type values.

10. Update `elbo_type` field default from `"trace"` to `"trace_elbo"` to align
    with the new ELBO_REGISTRY string keys. Update the docstring to list the
    3 valid elbo_type values.

11. Update the class-level docstring for both fields to reference the new
    string conventions.

Create `tests/test_elbo_variants.py` with comprehensive ELBO variant tests.
Use a minimal Pyro model (2-3 latent Normal sites with an observed site) that
runs fast (no ODE integration).

1. **Test valid combinations** (`test_valid_guide_elbo_combinations`):
   Use `@pytest.mark.parametrize` over all 14 valid (guide_type, elbo_type)
   pairs from the compatibility matrix:
   - auto_delta x {trace_elbo, tracemeanfield_elbo, renyi_elbo}
   - auto_normal x {trace_elbo, tracemeanfield_elbo, renyi_elbo}
   - auto_lowrank_mvn x {trace_elbo, renyi_elbo}
   - auto_mvn x {trace_elbo, renyi_elbo}
   - auto_iaf x {trace_elbo, renyi_elbo}
   - auto_laplace x {trace_elbo, renyi_elbo}

   For each: create guide via create_guide, run run_svi with 10 steps,
   assert loss list has 10 entries and final_loss is finite (not NaN/inf).
   Pass guide_type to run_svi so it can validate.

2. **Test rejected combinations** (`test_rejected_guide_elbo_combinations`):
   Parametrize over 4 rejected pairs:
   - auto_lowrank_mvn + tracemeanfield_elbo
   - auto_mvn + tracemeanfield_elbo
   - auto_iaf + tracemeanfield_elbo
   - auto_laplace + tracemeanfield_elbo

   For each: create guide, call run_svi, assert raises ValueError.
   Verify error message contains "mean-field" and the guide_type string.

3. **Test RenyiELBO uses alpha=0.5** (`test_renyi_elbo_alpha`):
   Run SVI with renyi_elbo, verify it completes without error.
   (Cannot easily inspect alpha after construction, so this is a smoke test.)

4. **Test RenyiELBO min particles** (`test_renyi_elbo_min_particles`):
   Run SVI with renyi_elbo and num_particles=1. Verify it completes
   (implementation should bump to 2 internally).

5. **Test AutoLaplace returns post-guide** (`test_auto_laplace_post_guide`):
   Create guide with guide_type='auto_laplace', run run_svi with
   guide_type='auto_laplace' and ~50 steps. Verify result dict has
   "guide" key and the returned guide is an AutoMultivariateNormal instance.

6. **Test default elbo_type is trace_elbo** (`test_default_elbo_type`):
   Call run_svi without elbo_type arg, verify it works (backward compatible).

7. **Test invalid elbo_type raises** (`test_invalid_elbo_type_raises`):
   Call run_svi with elbo_type='nonexistent', assert raises ValueError.

All tests use `from __future__ import annotations`, follow project conventions.
Mark the full parametrized combination test with `@pytest.mark.slow` if total

1. Create `tests/test_amortized_benchmark.py` with comprehensive benchmark tests.

This file tests the Phase 7 success criteria across both variants. Since full training
(10,000+ datasets) is too expensive for CI, use a scaled-down protocol that validates
the methodology and produces meaningful metrics.

**CI benchmark protocol (scaled down):**
- Training data: 200 simulated datasets (not 10,000)
- Training steps: 500 for spectral (fast), 200 for task (slow ODE)
- Held-out test: 20 datasets
- This validates the pipeline and methodology; full-scale training is done offline via the training script

**Helper functions (module-level or in conftest):**

`_train_amortized_guide(variant, n_train, n_steps, n_regions=3)`:
  - Generate training data using simulators
  - Create packer, fit standardization
  - Create guide (small config: n_transforms=3, hidden=[128,128])
  - Create wrapper model
  - Run SVI loop
  - Return trained guide, packer

`_run_per_subject_svi(variant, data, n_steps, n_regions=3)`:
  - Create standard model + AutoNormal guide
  - Run run_svi
  - Extract posterior params
  - Return median, ELBO

`_compute_metrics(guide, packer, test_data, test_params, per_subject_results)`:
  - For each test subject:
    - Amortized: guide.sample_posterior(data, n_samples=500), compute mean
    - Per-subject: already computed by _run_per_subject_svi
    - RMSE_amortized = sqrt(mean((A_mean_amort - A_true)^2))
    - RMSE_svi = sqrt(mean((A_mean_svi - A_true)^2))
  - Aggregate:
    - mean_rmse_ratio = mean(RMSE_amortized / RMSE_svi) across test subjects
    - Coverage: fraction of true params within amortized 90% CI
    - Amortization gap: (per_subject_ELBO - amortized_ELBO) / |per_subject_ELBO|

**Tests:**

- `test_spectral_amortized_vs_svi` (marked `@pytest.mark.slow`):
  - Train amortized guide on 200 spectral DCM datasets, 500 steps
  - Run per-subject SVI on 20 held-out datasets (500 steps each -- fast for spectral)
  - Compute RMSE ratio, coverage, amortization gap
  - Assert:
    - RMSE ratio < 2.0 (relaxed from 1.5 for small training set; full training targets 1.5)
    - Coverage in [0.50, 0.99] (relaxed from [0.85, 0.99] for small training)
    - Print all metrics for manual inspection
  - NOTE: Spectral DCM is ideal for this test because each SVI step is ~10ms. 200 datasets * 500 steps = ~1000 seconds = ~17 minutes. Manageable for a slow test.

- `test_task_amortized_elbo_direction` (marked `@pytest.mark.slow`):
  - Train amortized guide on 50 task DCM datasets (short: duration=60s, dt=1.0), 100 steps
  - Verify ELBO decreases (loss at step 100 < loss at step 0)
  - This is a lightweight validation that the task DCM amortized pipeline works end-to-end
  - Full task DCM benchmark requires hours of training and is done offline

- `test_inference_speed_both_variants`:
  - Create untrained guides for task (3-region) and spectral (3-region)
  - Fit dummy standardization
  - Time 1000 posterior samples each
  - Assert both < 1 second
  - This is a FAST test (no training needed, just forward pass)

- `test_rdcm_amortized_skip_documented`:
  - Simply verify that the docstring or a constant in `amortized_flow.py` or `amortized_wrappers.py` documents that rDCM amortized guide is intentionally skipped because the analytic VB posterior is exact for the conjugate model.
  - Check: `"rDCM" in amortized_wrappers.__doc__` or similar
  - This satisfies AMR-03 by explicit deferral.

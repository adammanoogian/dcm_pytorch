<objective>
Implement the amortized normalizing flow guide for task-based DCM (AMR-01). This is the
core deliverable: a conditional Neural Spline Flow (Zuko NSF) that takes BOLD observations
through a summary network and outputs approximate posterior samples over (A_free, C,
noise_prec) in a single forward pass -- no iterative optimization.

The main engineering challenge is the Pyro site-matching problem (07-RESEARCH.md Pitfall 1):
the flow outputs a single vector but the DCM model has multiple named sample sites. The
solution is a wrapper model that samples a single packed latent vector and deterministically
unpacks it, keeping Pyro's automatic ELBO machinery working.

Purpose: AMR-01 is the primary requirement -- demonstrate that normalizing flows can
amortize task DCM inference while maintaining accuracy close to per-subject SVI.

Output: `AmortizedFlowGuide` class, wrapper model, training script, and tests proving
ELBO convergence on task DCM data.
</objective>

1. Create `src/pyro_dcm/guides/amortized_flow.py` with the `AmortizedFlowGuide` class:

```
class AmortizedFlowGuide(nn.Module):
```

**Constructor args:**
- `summary_net: nn.Module` -- BoldSummaryNet or CsdSummaryNet
- `n_features: int` -- dimensionality of packed parameter vector
- `embed_dim: int = 128` -- summary network output dim (must match summary_net)
- `n_transforms: int = 5` -- number of NSF autoregressive transforms
- `n_bins: int = 8` -- number of spline bins
- `hidden_features: list[int] = [256, 256]` -- hidden layer sizes per transform
- `packer: TaskDCMPacker | SpectralDCMPacker` -- for standardize/unstandardize

**Constructor body:**
- Create `zuko.flows.NSF(features=n_features, context=embed_dim, bins=n_bins, transforms=n_transforms, hidden_features=hidden_features, passes=2)` -- coupling mode for fast sampling per 07-RESEARCH.md
- Store summary_net, flow, packer as attributes
- Store string `variant` ("task" or "spectral") for dispatch

**`forward` method (the Pyro guide function):**

The guide samples ONLY `_latent` from the flow. The wrapper model also samples ONLY `_latent` from the prior, then deterministically unpacks. Both model and guide have a single matching site -- this is the wrapper model pattern from 07-RESEARCH.md that solves the Pyro site-matching problem.

```python
def forward(self, observed_data, *args, **kwargs):
    pyro.module("summary_net", self.summary_net)
    pyro.module("flow", self.flow)
    embedding = self.summary_net(observed_data)
    flow_dist = self.flow(embedding)
    z_std = pyro.sample("_latent", ZukoToPyro(flow_dist))
    return z_std
```

**`sample_posterior` method (for inference-time use):**
```python
def sample_posterior(self, observed_data, n_samples=1000):
    """Draw posterior samples via forward pass (no SVI needed)."""
    self.eval()
    with torch.no_grad():
        embedding = self.summary_net(observed_data)
        flow_dist = self.flow(embedding)
        z_std = flow_dist.rsample((n_samples,))
        z = self.packer.unstandardize(z_std)
        return self.packer.unpack(z)  # dict of (n_samples, ...) tensors
```

**`amortized_task_dcm_model` function:**
- Signature: `(observed_bold, stimulus, a_mask, c_mask, t_eval, TR, dt, packer)`
- Instead of sampling A_free, C, noise_prec separately, samples a single `_latent` vector:
  ```python
  n_features = packer.n_features
  # Prior: standard normal in standardized space
  z_std = pyro.sample("_latent",
      dist.Normal(torch.zeros(n_features, dtype=torch.float64),
                  torch.ones(n_features, dtype=torch.float64)).to_event(1))
  z = packer.unstandardize(z_std)
  params = packer.unpack(z)
  A_free = params["A_free"] * a_mask
  A = pyro.deterministic("A", parameterize_A(A_free))
  C = params["C"] * c_mask
  noise_prec = params["noise_prec"].exp()  # stored in log-space in the packed vector
  ```
  The prior on `_latent` in standardized space is N(0,1) because the standardization was fit to match the data-generating distribution. This means N(0,1) approximately matches the actual prior after standardization.

- Then run the SAME forward model as `task_dcm_model` (CoupledDCMSystem, ODE integration, BOLD signal, likelihood).
- The forward model logic (ODE, BOLD, likelihood) should be directly reused from the existing functions, NOT reimplemented. Import and call the same functions.

**CRITICAL -- conditioning on observed data:** The wrapper model's BOLD likelihood site MUST condition on `observed_bold` using the `obs=` kwarg:
  ```python
  pyro.sample(
      "obs",
      dist.Normal(predicted_bold, noise_std).to_event(2),
      obs=observed_bold,
  )
  ```
  This is identical to how `task_dcm_model` conditions on observed data. Without `obs=observed_bold`, Pyro's ELBO will NOT compute the correct variational bound -- it would only evaluate KL(q||p) without the likelihood term. The `observed_bold` argument passed to the wrapper model function is forwarded directly to this `obs=` kwarg.

1. Create `scripts/train_amortized_guide.py`:

**Argparse CLI args:**
- `--variant` (task/spectral, required)
- `--data-path` (path to .pt training data file, required)
- `--n-regions` (default 3)
- `--n-inputs` (default 1, task DCM only)
- `--embed-dim` (default 128)
- `--n-transforms` (default 5)
- `--n-bins` (default 8)
- `--hidden-features` (default "256,256")
- `--lr` (default 1e-3)
- `--clip-norm` (default 10.0)
- `--num-particles` (default 16)
- `--batch-size` (default 128)
- `--n-epochs` (default 200)
- `--n-train-steps` (total SVI steps, default 50000, alternative to epochs)
- `--output-dir` (default `models/`)
- `--seed` (default 42)
- `--val-fraction` (default 0.1)

**Training loop:**
```python
# Load data
dataset = torch.load(data_path, weights_only=False)
data_list = dataset["data"]   # list of bold/csd tensors
params_list = dataset["params"]  # list of param dicts

# Split train/val
n_val = int(len(data_list) * val_fraction)
train_data, val_data = data_list[n_val:], data_list[:n_val]
train_params, val_params = params_list[n_val:], params_list[:n_val]

# Create packer and fit standardization on training params
packer = TaskDCMPacker(...)  # or SpectralDCMPacker
packer.fit_standardization(train_params)

# Create summary net and guide
summary_net = BoldSummaryNet(n_regions, embed_dim).double()
guide = AmortizedFlowGuide(summary_net, packer.n_features, ..., packer=packer).double()

# Create SVI with wrapper model
# The wrapper model receives observed_bold and conditions on it via obs= kwarg
svi = SVI(amortized_task_dcm_model, guide, ClippedAdam({"lr": lr, "clip_norm": clip_norm}),
          loss=Trace_ELBO(num_particles=num_particles, vectorize_particles=True))

# Training loop: iterate over dataset items
# batch_size controls how many dataset items are sampled per epoch.
# Each item is processed as an individual SVI step because each
# data point has a different observed_bold tensor.
for step in range(n_train_steps):
    # Sample random item from training data
    i = torch.randint(len(train_data), (1,)).item()
    bold_i = train_data[i]
    # SVI step: guide receives bold_i, wrapper model also receives
    # bold_i which it passes to obs= in the likelihood site
    loss = svi.step(bold_i, stimulus, a_mask, c_mask, t_eval, TR, dt, packer)
    epoch_loss += loss

    if step % 100 == 0:
        print(f"Step {step}: loss = {loss:.2f}")

    # Save checkpoint every 5000 steps
    if step % 5000 == 0 and step > 0:
        torch.save({...}, checkpoint_path)
```

IMPORTANT: For task DCM, each SVI step runs the full ODE, which is slow (~1-2s). For a training dataset of 10,000 with 200 epochs, this is infeasible on CPU. The training script should:
- Use `--n-train-steps` (total SVI steps, default 50000) for step-based training
- Sample random items from the dataset for each step
- Print progress every 100 steps
- Save checkpoint every 5000 steps
- For spectral DCM, the forward model is algebraic (no ODE), so epoch-based training is also supported

For task DCM specifically: the ODE forward pass in the wrapper model is the bottleneck. Consider using a COARSER dt (1.0 instead of 0.5) and shorter simulations for training speed. Document this trade-off.

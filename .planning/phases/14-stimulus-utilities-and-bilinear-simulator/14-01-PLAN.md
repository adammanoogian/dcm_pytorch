stick functions and `make_epoch_stimulus` for boxcar-shaped modulatory inputs) to the task
simulator module, plus one reusable helper (`merge_piecewise_inputs`) in the ODE integrator

input trajectories `u_mod(t)` in the Friston 2003 bilinear form.

### 1. `make_event_stimulus` in `src/pyro_dcm/simulators/task_simulator.py`

Add immediately AFTER the existing `make_block_stimulus` function (which lives around
lines 230-299 of task_simulator.py). Cite REF-001 context in the docstring and flag
Pitfall B12 (stick-function blur at rk4 mid-steps).

**Signature:**
```python
def make_event_stimulus(
    event_times: torch.Tensor | list[float],
    event_amplitudes: torch.Tensor | list[float] | list[list[float]],
    duration: float,
    dt: float,
    n_inputs: int | None = None,
    dtype: torch.dtype = torch.float64,
) -> dict[str, torch.Tensor]:
```

**Algorithm** (follow researcher pseudocode in 14-RESEARCH.md Section 1.3 verbatim):

1. Normalize `event_times` to a float tensor of shape `(n_events,)`.
2. Normalize `event_amplitudes`:
   - Scalar (`ndim == 0`): broadcast to `(n_events, n_inputs)`; default `n_inputs = 1`.
   - 1-D `(n_events,)`: column-0 amplitude, zero-pad to `(n_events, n_inputs)` if
     `n_inputs > 1`. Default `n_inputs = 1`.
   - 2-D `(n_events, n_inputs)`: used directly; infer `n_inputs = amps.shape[1]` if not
     provided, else validate that `amps.shape[1] == n_inputs`.
   - Else raise `ValueError` with explicit expected-vs-actual ndim.
3. Validate:
   - `dt > 0` and `duration > 0`, else `ValueError(f"dt={dt}, duration={duration} must be > 0")`.
   - All `event_times` in `[0, duration)`, else `ValueError` listing the offending values.
4. Sort by `event_times` ascending (save permutation; apply same permutation to `amps`).
5. Quantize onsets: `onset_idx = torch.round(event_times / dt).long()`, then
   `onset_t = onset_idx.to(dtype) * dt`.
6. **Raise `ValueError` if any two consecutive `onset_idx` values are equal** (per
   14-RESEARCH.md Section 3 edge-case R2: "events {i} and {i+1} quantize to the same dt
   grid index"; users who want two simultaneous events must supply one event with summed
   amplitudes).
7. Build the breakpoint list:
   - Initial row: `times=[0.0]`, `values=[zeros(n_inputs)]`.
   - For each event `i`:
     - Append `(onset_t[i], amps[i])`.
     - Let `t_off = onset_t[i] + dt`. If `t_off <= duration`, append
       `(t_off, zeros(n_inputs))`. If `t_off > duration`, emit a single `UserWarning`
       (only once per call) that the tail of event `i` was truncated at end-of-sim,
       and skip the off-transition.
8. Stack into tensors; return `{"times": times, "values": values}` where `times` has
   shape `(K,)` and `values` has shape `(K, n_inputs)`.

**Docstring MUST include:**
- NumPy-style sections: Parameters, Returns, Raises, Warns, Notes, Examples.
- Citation to REF-001 context (driving input `u(t)` in Eq. 1).
- Explicit warning block: "For bilinear modulatory inputs (`stimulus_mod` in
  `simulate_task_dcm`), prefer `make_epoch_stimulus` (boxcars). Stick functions are
  blurred to ~2x width by rk4 mid-step sampling (Pitfall B12)."
- Example showing a 3-event design at `dt=0.01` with scalar amplitude.

**Example code for docstring:**
```python
>>> stim = make_event_stimulus(
...     event_times=[1.0, 3.0, 5.0],
...     event_amplitudes=[0.5, 1.0, 0.7],
...     duration=10.0,
...     dt=0.01,
... )
>>> stim["times"].shape
torch.Size([7])
>>> stim["values"].shape
torch.Size([7, 1])
```

### 2. `make_epoch_stimulus` in `src/pyro_dcm/simulators/task_simulator.py`

Add immediately AFTER `make_event_stimulus`. Cite Pitfall B12 as the *reason this is
the preferred primitive for modulators*.

**Signature:**
```python
def make_epoch_stimulus(
    event_times: torch.Tensor | list[float],
    event_durations: torch.Tensor | list[float] | float,
    event_amplitudes: torch.Tensor | list[float] | list[list[float]],
    duration: float,
    dt: float,
    n_inputs: int | None = None,
    dtype: torch.dtype = torch.float64,
) -> dict[str, torch.Tensor]:
```

**Algorithm** (follow 14-RESEARCH.md Section 2.2 pseudocode):

1. Normalize `event_times`, `event_amplitudes` identically to `make_event_stimulus`.
2. Normalize `event_durations`:
   - Scalar: `expand_as(event_times).clone()`.
   - 1-D: must match `event_times.shape[0]`, else `ValueError`.
   - All values must be `> 0`, else `ValueError("event_durations must all be > 0")`.
3. Validate `event_times` in `[0, duration)` as before.
4. Compute quantized on/off times:
   - `t_on = torch.round(event_times / dt) * dt`
   - `t_off_raw = t_on + event_durations`
   - `t_off = torch.clamp(t_off_raw, max=duration)`
   - If any `t_off_raw > duration`: emit **one** `UserWarning("Some epochs clipped to
     duration")`.
5. Build the delta-amp event list:
   - For each event `i`: add `(t_on[i], +amps[i])` and `(t_off[i], -amps[i])` to the
     event list.
   - Sort event list by time ascending.
6. Sweep the event list with a running `current = zeros(n_inputs)`:
   - For each `(t, delta)`: `current = current + delta`.
   - If `times_list[-1] == t` (exact float equality on a shared dt grid; acceptable
     per researcher Section 5.2): replace `values_list[-1]` with `current.clone()`
     (collapse duplicate-time breakpoints).
   - Else append `(t, current.clone())` as a new breakpoint.
   - **Overlap detection:** if at any time step `current.abs().sum() > max_individual_amp +
     1e-12` (i.e., two epochs simultaneously active in the same column), emit **one**
     `UserWarning("Overlapping epochs detected; amplitudes are summed. If you want
     override semantics, pre-flatten events.")` on the first such detection. Set a
     flag so the warning fires at most once per call.
7. Return `{"times": times, "values": values}` with shapes `(K,)` and `(K, n_inputs)`.

**Overlap semantics docstring (locked per L1):**
> When two epochs overlap in time, their amplitudes **sum** (e.g., two overlapping
> unit-amplitude epochs produce amplitude 2 during the overlap window). A single
> `UserWarning` is emitted at construction time. If override-semantics are desired
> (event `i+1` cancels event `i`'s tail), users must pre-flatten the event schedule
> themselves. This decision matches the bilinear DCM neural equation
> `A_eff(t) = A + Σ_j u_j(t) · B_j`, where simultaneous modulators already superpose.

**Docstring MUST include:**
- NumPy-style sections: Parameters, Returns, Raises, Warns, Notes, Examples.
- Explicit statement: "Preferred primitive for modulatory inputs (`stimulus_mod`) in
  bilinear DCM. Unlike `make_event_stimulus` (stick functions), boxcars are
  dt-invariant under rk4 mid-step sampling (Pitfall B12)."
- Example showing a single 40s epoch at amplitude 1.0.

### 3. `merge_piecewise_inputs` in `src/pyro_dcm/utils/ode_integrator.py`

Add at the END of the file (after `make_initial_state` / existing helpers). This helper
is PUBLIC so Phase 15's Pyro model can import it without crossing a `simulators/`
boundary.

**Signature:**
```python
def merge_piecewise_inputs(
    drive: PiecewiseConstantInput,
    mod: PiecewiseConstantInput,
) -> PiecewiseConstantInput:
```

**Algorithm** (follow 14-RESEARCH.md Section 5.2 pseudocode):

1. Read `t_drive = drive.times`, `t_mod = mod.times`, `v_drive = drive.values`,
   `v_mod = mod.values`. Read `dtype = v_drive.dtype` and `device = v_drive.device`.
2. **Validate dtype/device match** between drive and mod. If `v_mod.dtype != dtype` or
   `v_mod.device != device`, raise `ValueError` with explicit expected-vs-actual
   (mitigates risk R3 in 14-RESEARCH.md Section 10.2). Do NOT auto-cast silently.
3. Compute sorted union: `all_times = torch.cat([t_drive, t_mod])`;
   `merged_times = torch.unique(all_times, sorted=True)`.
4. Allocate `merged_values = torch.empty((K, M+J), dtype=dtype, device=device)`
   where `K = merged_times.shape[0]`, `M = v_drive.shape[1]`, `J = v_mod.shape[1]`.
5. For each `k, t_k` in `enumerate(merged_times)`:
   - `merged_values[k, :M] = drive(t_k.detach())`
   - `merged_values[k, M:] = mod(t_k.detach())`
6. Return `PiecewiseConstantInput(merged_times, merged_values)`.

**Docstring MUST include:**
- NumPy-style sections: Parameters, Returns, Raises, Notes, Examples.
- Note on correctness: "At any query time `t*`, the merged input returns
  `[drive(t*), mod(t*)]` concatenated along the column axis. Because the merged
  breakpoint set contains every discontinuity of both inputs and
  `PiecewiseConstantInput.__call__` is left-closed, the merged breakpoint values
  at `t_k` equal `drive(t_k)` concatenated with `mod(t_k)`."
- Note on complexity: "O(K log K) where K = len(merged_times); typical DCM runs
  (K1 ~ 20 blocks, K2 ~ 40 events) complete in microseconds."
- Example showing a 2-column drive merged with a 1-column mod producing a 3-column
  widened input.
